# 业务需求工作量评估系统 - 项目说明文档

## 一、项目背景

### 1.1 项目概述
业务需求工作量评估系统是一个基于Agent智能体的智能化需求评估工具，旨在解决传统工作量评估过程中存在的以下痛点：
- 人工拆分效率低、耗时长
- 评估标准不统一、偏差大
- 缺乏科学的方法论支撑
- 评估结果难以追溯和复用

### 1.2 核心价值
- ✅ **自动化**：自动解析需求文档、识别系统、拆分功能点
- ✅ **智能化**：基于AI大模型的智能分析
- ✅ **标准化**：统一的功能点拆分标准和评估方法
- ✅ **可追溯**：完整的评估过程记录和结果输出
- ✅ **可集成**：提供RESTful API，易于集成到DevOps系统

### 1.3 应用场景
- 需求评审阶段的工作量预估
- 项目投标时的工作量评估
- 项目管理中的资源规划
- DevOps系统的需求评估模块

## 二、核心功能

### 2.1 功能清单

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|--------|
| 文档解析 | 自动解析.docx格式需求文档 | P0 |
| 系统识别 | 智能识别需求涉及的所有系统 | P0 |
| 功能点拆分 | 按系统维度拆分功能点 | P0 |
| 工作量估算 | Delphi专家评估法 | P0 |
| 报告生成 | 生成Excel评估报告 | P0 |
| Web界面 | 文件上传、进度查看、报告下载 | P1 |
| API接口 | RESTful API接口 | P0 |
| 配置管理 | 专家权重、COSMIC规则配置 | P2 |

### 2.2 业务流程

```
1. 用户上传需求文档（.docx格式）
   ↓
2. 系统自动解析"需求内容说明"章节
   ↓
3. 系统识别Agent识别涉及系统
   ↓
4. 功能点拆分Agent按系统拆分功能点
   ↓
5. 工作量估算Agent进行Delphi估算
   ↓
6. 报告生成Agent生成Excel报告
   ↓
7. 用户下载报告
```

## 四、功能点拆分规则

### 4.1 拆分原则

1. **系统维度优先**：按涉及系统进行一级拆分
2. **功能模块为辅**：系统内按功能模块进行二级拆分
3. **粒度适中**：单个功能点0.5-5人天
4. **独立可估算**：每个功能点可独立估算工作量
5. **依赖清晰**：明确标注功能点间的依赖关系

### 4.2 拆分方法

根据需求特征选择拆分方法：

| 需求类型 | 特征描述 | 推荐方法 |
|---------|---------|---------|
| 数据处理型 | 单角色、无流程、功能独立 | 按业务功能模块拆分 |
| 流程驱动型 | 多角色、强流程 | 按用户角色+场景拆分 |
| 界面优化型 | 单角色、无流程、独立优化点 | 按业务功能模块拆分 |
| 全新产品型 | 需求模糊、需快速验证 | 按MVP阶段拆分 |

### 4.3 复杂度评估标准

**高复杂度**：
- 涉及多系统集成
- 复杂算法或业务逻辑
- 核心业务功能
- 高风险或新技术

**中复杂度**：
- 标准业务逻辑
- 一定复杂度
- 常规功能

**低复杂度**：
- 简单增删改查
- 配置管理
- 查询展示

## 五、Delphi专家评估法

### 5.1 方法说明

Delphi法是一种结构化的专家 forecasting 方法，通过多轮匿名调查和反馈，使专家意见逐步收敛。

### 5.2 实施步骤

1. **第一轮**：专家独立估算每个功能点的工作量
2. **反馈**：展示统计结果（最大值、最小值、平均值、标准差）
3. **第二轮**：专家参考反馈，调整自己的估算
4. **第三轮**：重复反馈-调整过程（通常2-3轮）
5. **收敛**：取加权平均值作为最终估算

### 5.3 专家权重配置

系统支持配置不同专家的权重：
- 核心专家：权重1.0
- 普通专家：权重0.8
- 权重根据专家经验和历史准确度确定

## 六、系统配置

### 6.1 环境变量

```bash
# 阿里云大模型配置
DASHSCOPE_API_KEY=sk-xxxxx
DASHSCOPE_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen-turbo
LLM_TEMPERATURE=0.7
LLM_MAX_TOKENS=4000

# 应用配置
DEBUG=false
HOST=0.0.0.0
PORT=8000
WORKERS=4

# Delphi配置
DELPHI_ROUNDS=3
```

### 6.2 COSMIC功能点配置

系统内置COSMIC全功能点算法，可配置数据移动权重：
- E (Entry): 入口数据移动权重 = 2
- R (Read): 读数据移动权重 = 2
- W (Write): 写数据移动权重 = 2
- X (Exit): 出口数据移动权重 = 1

## 八、常见问题

### Q1: 支持哪些文档格式？
A: 目前仅支持.docx格式。后续可扩展支持.pdf、.txt等格式。

### Q2: 如何提高评估准确性？
A:
1. 提供高质量的需求文档
2. 定期优化提示词模板
3. 根据实际数据调整专家权重
4. 收集反馈，持续改进模型

### Q3: 能否自定义拆分规则？
A: 可以。通过修改`backend/prompts/prompt_templates.py`中的提示词模板来自定义拆分规则。
