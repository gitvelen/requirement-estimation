## 一、概述

本文档将系统升级任务分解为可执行的开发任务，涵盖前端、后端、数据库、测试等各个方面。

### 1.0 本期范围裁剪（2026-01-30）

本期聚焦“知识库尽快可用”，以下任务延期到下一期：

- 部署：DEPLOY-003（K8s配置）、DEPLOY-004（CI/CD配置）
- 持久化与缓存：DB-001~DB-006、BE-002（数据库连接）、BE-003（Redis）
- 业务文件对象存储：BE-004、INT-003（MinIO集成）
- 服务化向量库：INT-004（Milvus集成）
- 生产化中间件：BE-007（日志中间件）、BE-008（统一异常处理）

本期文件处理落地策略：需求文档/上传文件走本地 `uploads/` 挂载目录（不接入 MinIO 作为业务文件存储）。
本期知识库落地策略：使用本地文件向量库（`data/knowledge_store.json`），后续再切换 Milvus。

### 1.1 任务分类

| 类别 | 说明 | 优先级 |
|------|------|--------|
| P0 | 核心功能，必须完成 | 高 |
| P1 | 重要功能，应该完成 | 中 |
| P2 | 增强功能，可以延后 | 低 |

- [ ] 用户可以正常登录
- [ ] 项目经理可以上传文档并创建任务
- [ ] AI可以正确解析文档并识别功能点
