# 知识库功能验收文档

## 验收目的

验证知识库功能是否完整可用，包括：
- （可选）Milvus向量数据库集成
- 系统知识导入（DOCX/PPTX，按主系统维度）
- 知识检索
- 统计信息
- 效果评估指标

## 验收标准

### 1. 部署验收

#### 1.1 Docker一键部署

**验收标准**：
- ✅ 使用单个命令启动所有服务
- ✅ Milvus服务自动启动（--profile milvus）
- ✅ 服务间网络通信正常
- ✅ 数据持久化到Docker卷

**验收命令**：
```bash
# 启动所有服务（包括Milvus）
docker-compose --profile milvus up -d

# 查看服务状态
docker-compose ps
```

**预期结果**：
```
NAME                      STATUS
milvus-etcd               Up (healthy)
milvus-minio              Up (healthy)
milvus-standalone         Up (healthy)
requirement-backend       Up (healthy)
requirement-frontend      Up
```

#### 1.2 健康检查

**验收标准**：
- ✅ Milvus健康检查通过
- ✅ Backend API健康检查通过
- ✅ 知识库API健康检查通过

**验收命令**：
```bash
# Milvus健康检查
curl http://localhost:9091/healthz

# Backend健康检查
curl http://localhost/api/v1/health

# 知识库健康检查
curl http://localhost/api/v1/knowledge/health
```

**预期结果**：
- 所有HTTP状态码为200
- 返回状态为"healthy"

### 2. 功能验收

#### 2.1 知识库导入功能

**验收标准**：
- ✅ 支持DOCX格式智能提取（system_profile）
- ✅ 支持PPTX格式智能提取（system_profile）
- ✅ 导入时必须选择主系统（一个主系统一套知识库）
- ✅ 导入结果显示成功/失败数量
- ✅ 错误信息清晰明确

**验收步骤**：
1. 访问知识库管理页面：`http://your-domain/knowledge`
2. 选择一个“主系统”
3. 点击下载 DOCX 或 PPTX 模板
4. 填写1条测试数据（系统名称、业务目标、技术栈等）
5. 上传 DOCX/PPTX 文件
6. 查看导入结果

**预期结果**：
```
导入完成！成功 2 条，失败 0 条
```

#### 2.2 知识库检索功能

**验收标准**：
- ✅ 支持关键词检索
- ✅ 返回相似度排序的结果
- ✅ 按主系统维度检索（system_name 必填）
- ✅ 检索结果可展开查看详情

**验收步骤**：
1. 在知识库管理页面搜索框输入关键词
2. 点击"搜索"按钮
3. 查看检索结果列表
4. 点击某条记录查看详情

**预期结果**：
- 显示相关知识列表
- 相似度从高到低排序
- 可展开查看完整内容和元数据

#### 2.3 统计信息

**验收标准**：
- ✅ 显示当前主系统知识总数
- ✅ 显示索引类型和度量方式
- ✅ 数据实时更新

**验收步骤**：
1. 访问知识库管理页面
2. 查看顶部的统计卡片

**预期结果**：
```
知识总数: 10 条
索引类型: IVF_FLAT
```

#### 2.4 效果评估指标

**验收标准**：
- ✅ 显示检索命中率
- ✅ 显示平均相似度
- ✅ 显示案例采纳率
- ✅ 显示质量对比（使用 vs 未使用知识库）
- ✅ 支持手动刷新

**验收步骤**：
1. 在知识库管理页面查看"知识库效果评估"卡片
2. 点击"刷新"按钮
3. 查看各项指标

**预期结果**：
- 显示所有评估指标
- 数据随使用累积更新
- 刷新后数据更新

### 3. 性能验收

#### 3.1 响应时间

**验收标准**：
- ✅ 知识导入 < 10秒（10条数据）
- ✅ 知识检索 < 2秒
- ✅ 统计信息 < 1秒

**验收命令**：
```bash
# 使用验收脚本测试
python scripts/knowledge_acceptance.py
```

#### 3.2 并发支持

**验收标准**：
- ✅ 支持10个并发导入请求
- ✅ 支持50个并发检索请求

### 4. 数据质量验收

#### 4.1 系统知识数据

**验收标准**：
- ✅ 系统名称与 system_list.md 一致
- ✅ 包含完整的字段（业务目标、技术栈等）
- ✅ 数据格式规范

**验收步骤**：
1. 导入系统知识 DOCX/PPTX
2. 检索某个系统
3. 验证返回的数据完整性

### 5. 集成验收

#### 5.1 Agent集成

**验收标准**：
- ✅ 系统识别Agent能够检索系统知识
- ✅ 功能拆分Agent能够检索系统知识（系统边界/核心功能/技术栈）
- ✅ 日志中显示知识库使用情况

**验收步骤**：
1. 导入一些测试数据到知识库
2. 上传需求文档进行评估
3. 查看后端日志

**预期日志**：
```
[系统识别] 正在检索相关系统知识...
[系统识别] 检索到 2 条相关系统知识
[功能拆分] 正在检索【支付中台】的系统知识...
[功能拆分] 检索到 3 条相关系统知识
```

#### 5.2 降级机制

**验收标准**：
- ✅ Milvus不可用时自动降级
- ✅ 不影响基础评估功能
- ✅ 日志中显示降级信息

**验收步骤**：
1. 停止Milvus服务
2. 上传需求文档进行评估
3. 验证评估功能正常

**预期日志**：
```
知识库功能未启用
[系统识别] 未检索到相关系统知识
```

## 自动化验收脚本

项目提供了自动化验收脚本：`scripts/knowledge_acceptance.py`

### 使用方法

```bash
# 基础用法
python scripts/knowledge_acceptance.py

# 指定API地址
python scripts/knowledge_acceptance.py --base-url http://your-server-ip

# 查看帮助
python scripts/knowledge_acceptance.py --help
```

### 测试内容

脚本会自动执行以下测试：

1. **Milvus连接测试**
   - 检查Milvus容器状态
   - 检查Milvus健康状态
   - 检查知识库API健康状态

2. **统计信息测试**
   - 获取知识库统计信息
   - 验证数据完整性

3. **效果评估测试**
   - 获取评估指标
   - 验证数据合理性

4. **导入测试**
   - 导入DOCX/PPTX测试数据（需携带 system_name）
   - 验证导入结果

5. **检索测试**
   - 执行知识检索
   - 验证检索结果

## 验收结论

### 合格标准

知识库功能验收合格需要满足：

1. **部署验收**：所有项目必须通过
2. **功能验收**：至少90%的功能测试通过
3. **性能验收**：所有性能指标达标
4. **集成验收**：Agent集成正常，降级机制有效

### 验收流程

1. **环境准备**：启动Milvus和应用服务
2. **功能测试**：执行所有功能验收测试
3. **性能测试**：执行性能验收测试
4. **集成测试**：执行集成验收测试
5. **生成报告**：运行自动化验收脚本
6. **签署确认**：确认验收通过

### 验收不合格处理

如果验收不合格：

1. 记录失败项
2. 分析失败原因
3. 修复问题
4. 重新验收
5. 生成新的验收报告
