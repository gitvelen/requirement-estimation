# 前后端独立部署指南

本文档说明如何将前后端分别部署到不同服务器。

---

## 目录结构

```
requirement-estimation-system/
├── docker-compose.backend.yml      # 后端部署配置
├── docker-compose.frontend.yml     # 前端部署配置
├── .env.backend                     # 后端环境变量（需创建）
├── .env.frontend                   # 前端环境变量（需创建）
└── frontend/
    ├── Dockerfile                  # 前端镜像构建
    └── nginx-remote.conf           # Nginx 配置（代理到远程后端）
```

---

## 一、后端部署

### 1. 准备工作

在**后端服务器**上：

```bash
# 复制项目文件到服务器
scp -r requirement-estimation-system/ user@backend-server:/opt/

# SSH 登录后端服务器
ssh user@backend-server
cd /opt/requirement-estimation-system
```

### 2. 配置环境变量

```bash
# 复制示例配置
cp .env.backend.example .env.backend

# 编辑配置，修改以下关键参数
nano .env.backend
```

**必填配置**：
```bash
DASHSCOPE_API_KEY=your_dashscope_api_key
JWT_SECRET=random_secret_string_here
ADMIN_API_KEY=your_admin_api_key
```

### 3. 启动后端

```bash
# 构建并启动
docker-compose -f docker-compose.backend.yml up -d

# 查看日志
docker-compose -f docker-compose.backend.yml logs -f

# 检查健康状态
curl http://localhost:443/api/v1/health
```

### 4. 防火墙配置

```bash
# 开放 443 端口
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --reload
```

## 二、前端部署

### 1. 准备工作

在**前端服务器**上：

```bash
# 复制项目文件到服务器（只需要 frontend 目录和配置文件）
scp -r requirement-estimation-system/frontend user@frontend-server:/opt/
scp requirement-estimation-system/docker-compose.frontend.yml user@frontend-server:/opt/
```

### 2. 配置后端地址

修改 `frontend/nginx-remote.conf`：

```bash
ssh user@frontend-server
cd /opt/frontend

# 编辑 nginx 配置，将 BACKEND_SERVER_PLACEHOLDER 替换为实际后端地址
nano nginx-remote.conf
```

**修改位置**（第 36 行）：
```nginx
proxy_pass https://YOUR_BACKEND_IP:443;
```

**示例**：
```nginx
proxy_pass http://192.168.1.100:443;          # 内网IP
# 或
proxy_pass https://api.yourdomain.com:443;    # 域名
```

### 3. 启动前端

```bash
# 构建并启动
docker-compose -f docker-compose.frontend.yml up -d

# 查看日志
docker-compose -f docker-compose.frontend.yml logs -f
```

### 4. 验证部署

访问前端服务器 IP：
```
http://FRONTEND_SERVER_IP
```

## 三、常见问题

### 1. 前端无法连接后端

**检查项**：
- 后端服务器防火墙是否开放 443 端口
- nginx-remote.conf 中的 `proxy_pass` 地址是否正确
- 后端服务是否正常运行：`docker logs requirement-backend`

### 2. HTTPS/SSL 混合内容问题

如果后端使用 HTTPS，前端也需要使用 HTTPS：

```nginx
# nginx-remote.conf 中添加
proxy_ssl_verify off;  # 如果使用自签名证书
```

### 3. CORS 跨域问题

后端需要允许前端域名的跨域请求，检查后端 CORS 配置。

## 四、端口规划建议

| 服务器 | 服务 | 端口 | 说明 |
|--------|------|------|------|
| 后端服务器 | Backend | 443 | API 服务 |
| 前端服务器 | Frontend | 80 | Web 界面 |

## 五、快速测试命令

```bash
# 后端健康检查
curl http://BACKEND_IP:443/api/v1/health

# 前端访问测试
curl http://FRONTEND_IP:80

# 查看容器状态
docker ps -a
```
