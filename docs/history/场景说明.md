# 需求评估系统 - 场景说明

> 目的：沉淀系统“可能出现的各个场景”及其边界，用于需求沟通、交互设计、接口设计与验收用例编写。  
> 约定：场景编号稳定（SC-xx），后续新增场景只追加编号，不改既有编号含义。

## 1. 角色与对象

### 1.1 角色

- 管理员（admin）：系统配置、用户管理、查看所有任务、分配专家、证据等级规则配置等。
- 项目经理（manager）：发起评估任务、编辑与提交评估、维护证据/画像/ESB/代码扫描等索引数据。
- 专家（expert）：参与任务评估，多轮盲评；可在线预览与任务关联的证据材料（不提供下载）。

### 1.2 核心对象

- 任务 Task：一次需求评估的载体（需求文本、系统识别分层、拆分结果、估算、专家轮次、报告版本、审计留痕等）。
- 系统识别分层：`selected / maybe / questions`（确定系统/不确定系统/最小澄清问题集）。
- 证据库（A层）：非结构化材料（DOCX/PPTX/PDF）导入、切块、检索、预览。
- 系统画像（B层）：结构化的业务能力边界摘要（草稿/发布），字段可回链证据依据。
- ESB 索引（I层）：集成点明细（提供方/调用方/交易/场景/状态），用于提示与澄清问题候选。
- 代码扫描（C层）：从 Spring Boot/Java 代码产出能力目录（事实层），用于召回与归属校验提示（不扩范围）。
- 证据等级：E0~E3（用于阻断与风险提示，规则由 admin 配置；任务内可被 manager 覆盖并留痕）。

## 2. 场景分类

为便于沟通与设计，本说明按“关注点”对场景做分类（分类编号稳定）：

| 分类 | 名称 | 说明 |
|---|---|---|
| CAT-A | 账号与权限 | 登录、鉴权、账号治理与安全边界 |
| CAT-B | 任务评估主链路 | 上传→识别→拆分→编辑→提交→分配→评估→报告→归档 |
| CAT-C | 知识/证据/索引 | 证据库A、画像B、ESB索引I、代码扫描C、系统知识等 |
| CAT-D | 规则与治理 | 证据等级规则、任务内覆盖、规则变更审计等 |
| CAT-E | 指标与通知 | 效果统计、通知、未读与消息处理 |
| CAT-F | 异常与运维排障 | AI失败/降级、权限失败、端口冲突、502等排障场景 |
| CAT-G | 配置管理（基础） | 系统清单、COSMIC规则、用户管理等基础配置与批量导入 |

## 4. 场景详述

### SC-01 首次访问与登录

- 分类：CAT-A（账号与权限）
- 角色：admin / manager / expert
- 触发：用户访问 `http://<HOST>/`
- 前置：服务已部署可访问（80端口可用）
- 主流程：
  1. 未登录用户访问任意业务页 → 跳转到登录页
  2. 输入用户名/密码登录 → 返回 token 与用户信息（角色）
  3. 登录后按角色跳转到默认页（任务管理/个人中心等）
- 产出：JWT token、本地登录态
- 失败/异常：
  - 用户名/密码错误 → 提示重试
  - 账号已禁用 → 提示不可登录

### SC-02 账号启停用/密码修改

- 分类：CAT-A（账号与权限）
- 角色：admin（启停用/用户管理）、全角色（修改密码）
- 主流程：
  - admin 在用户管理中创建/编辑用户、调整角色、启用/禁用账号
  - 用户在个人中心修改密码（验证旧密码）
- 产出：用户数据更新、活动日志/审计记录
- 风险点：生产环境需避免默认密码长期存在

### SC-03 项目经理创建任务（上传需求）

- 分类：CAT-B（任务评估主链路）
- 角色：manager
- 前置：
  - 已登录
  - （可选）已维护系统清单/画像/证据/ESB/能力目录中的任意一类，以提升准确性
- 主流程：
  1. 进入上传页面上传需求文档
  2. 系统抽取需求文本并触发 AI 编排
  3. 生成任务并进入“处理中”
- 产出：任务、需求原文、AI中间结果（系统识别分层/拆分/估算等）

### SC-04 AI处理失败与重试/降级

- 分类：CAT-F（异常与运维排障）
- 角色：manager / admin
- 触发：任务AI处理失败（模型超时/Key缺失/解析异常等）
- 主流程：
  - 展示失败原因（可理解的错误信息）
  - 允许继续进入编辑页（人工补齐关键信息），或在页面提供重试入口（若系统提供）
- 产出：任务仍可进入“人工修正链路”
- 典型原因：
  - 未配置模型 Key 导致 embedding/LLM 不可用
  - 文档格式不符合解析规则

### SC-05 系统识别分层复核与澄清问题整理

- 分类：CAT-B（任务评估主链路）
- 角色：manager
- 主流程：
  1. 在编辑页查看 `selected/maybe/questions`
  2. 对 selected 系统逐个复核，必要时改名/删除
  3. 对 maybe 系统判断是否需要新增为系统Tab
  4. 将 questions 用于需求澄清或补充材料收集
- 产出：系统识别分层定稿（为后续拆分与评估提供边界）

### SC-06 新增系统Tab并自动拆分

- 分类：CAT-B（任务评估主链路）
- 角色：manager
- 触发：从候选系统一键新增系统Tab，或手动新增
- 主流程：
  - 新增系统Tab → 系统自动触发拆分（失败也保留空Tab）
- 产出：新系统的功能点列表或空结果（提示原因）

### SC-07 单系统重拆（rebreakdown）

- 分类：CAT-B（任务评估主链路）
- 角色：manager
- 触发：某系统拆分质量不满意，需覆盖重拆
- 主流程：
  - 对指定系统发起重拆 → 覆盖当前系统拆分结果
- 失败兜底：重拆失败则保留原结果，提示错误原因

### SC-08 功能点编辑（增删改）与审计

- 分类：CAT-B（任务评估主链路）
- 角色：manager
- 主流程：
  1. 编辑功能点字段（名称、描述、输入/输出/依赖、类型、备注等）
  2. 新增功能点（需明确系统归属）
  3. 删除功能点（保留审计痕迹/可回放）
  4. 保存后生成修改记录，用于报告/复核
- 产出：结构化功能点、修改记录、（可选）重新生成报告版本

### SC-09 提交给管理员分配专家

- 分类：CAT-B（任务评估主链路）
- 角色：manager
- 主流程：
  - 编辑完成后提交任务进入待分配
- 产出：任务状态变更、通知（如系统启用）

### SC-10 管理员分配专家与校验

- 分类：CAT-B（任务评估主链路）
- 角色：admin
- 主流程：
  1. 在任务列表选择任务
  2. 选择 3 位专家分配
  3. 系统校验专家必须为：系统用户、角色为 expert、账号启用、在岗
- 产出：分配记录、邀请/通知、任务进入评估中

### SC-11 专家评估（草稿/提交/撤回）

- 分类：CAT-B（任务评估主链路）
- 角色：expert
- 主流程：
  1. 进入“我的评估”列表打开任务
  2. 对功能点逐条填写人天，可保存草稿
  3. 提交评估进入本轮完成状态
  4. 在允许条件下可撤回并重新提交
- 产出：专家评估记录（按轮次）

### SC-12 多轮评估触发（偏离度阈值）

- 分类：CAT-B（任务评估主链路）
- 角色：expert / admin（推进）
- 触发：AI预估 vs 专家均值偏离度超过阈值（例如 20%）
- 主流程：
  - 系统计算偏离度并生成高偏离列表
  - 触发下一轮（最多轮次可配置），专家重新评估
- 产出：多轮评估结果、偏离度统计

### SC-13 生成报告与版本管理

- 分类：CAT-B（任务评估主链路）
- 角色：admin / manager
- 主流程：
  1. 评估完成后生成报告（Excel为主）
  2. 当任务数据被再次修改或新一轮评估完成时，生成新报告版本
  3. 支持查看/下载报告版本列表
- 产出：报告文件、版本元数据

### SC-14 任务归档/删除与保留策略

- 分类：CAT-B（任务评估主链路）
- 角色：admin / manager
- 主流程：
  - 归档：任务从活跃列表进入归档（可追溯）
  - 删除：彻底删除（需谨慎，可能影响审计与复盘）
  - 保留策略：超过保留天数的完成/失败任务自动清理（按配置）

### SC-15 证据库导入与检索

- 分类：CAT-C（知识/证据/索引）
- 角色：manager
- 主流程：
  1. 按系统导入证据材料（DOCX/PPTX/PDF），填写信任等级与日期等元数据
  2. 系统抽取文本并切块入库，写入向量索引
  3. 在任务分析或独立检索中按系统过滤检索证据块
- 产出：证据块索引、证据统计（按系统/类型等）
- 失败/异常：
  - 扫描版PDF无可抽取文本 → 导入失败并给出可理解提示

### SC-16 专家在线预览证据（权限边界）

- 分类：CAT-C（知识/证据/索引）
- 角色：expert
- 前置：专家必须参与该任务；预览请求需携带 `task_id`
- 主流程：
  - 在评估/详情页点击预览证据
  - PDF 以内嵌方式预览；其它类型返回抽取文本预览（提示版式不一致）
- 安全边界：
  - 不提供下载
  - 专家不可浏览“全库证据”，仅限任务关联

### SC-17 系统画像维护（草稿/发布）

- 分类：CAT-C（知识/证据/索引）
- 角色：manager / admin
- 主流程：
  1. 维护系统画像字段（业务目标、核心能力、范围内/外、对象、术语、集成点、关键约束等）
  2. 草稿态仅用于提示，不作为硬约束
  3. 发布后作为强约束注入系统识别与拆分（并可回链证据引用）
- 产出：画像数据、发布态约束信号

### SC-18 ESB 索引导入/检索/统计

- 分类：CAT-C（知识/证据/索引）
- 角色：manager
- 主流程：
  - 导入 ESB 明细（CSV/XLSX），系统建立索引与统计
  - 检索支持范围切换（提供方/调用方/both），默认过滤仅“正常使用”
  - 用于集成点摘要与澄清问题候选（不作为功能清单）
- 产出：ESB索引、检索结果、统计

### SC-19 代码扫描运行/查看/提交结果

- 分类：CAT-C（知识/证据/索引）
- 角色：manager / admin
- 主流程：
  1. 提交扫描任务（repo_path、系统信息）
  2. 查看扫描状态与结果
  3. 提交结果进入能力目录（用于召回与归属校验提示）
- 产出：扫描作业、能力目录（事实层）

### SC-20 证据等级规则配置与变更日志

- 分类：CAT-D（规则与治理）
- 角色：admin
- 主流程：
  - 查看当前证据等级规则
  - 修改并保存规则
  - 系统记录变更日志（审计可追溯）
- 产出：规则版本、变更日志

### SC-21 任务内证据等级覆盖修正（需备注）

- 分类：CAT-D（规则与治理）
- 角色：manager
- 主流程：
  - 对某任务内某系统 evidence_level 进行覆盖修正
  - 必须填写备注，系统记录审计
- 产出：任务内覆盖记录（仅影响当前任务）

### SC-22 AI效果报告查看（专家匿名口径）

- 分类：CAT-E（指标与通知）
- 角色：admin / manager / expert
- 主流程：
  - admin/manager：按系统/时间/轮次等过滤查看指标快照与汇总
  - expert：默认仅可查看“我参与任务”的匿名汇总口径（去除经理信息等敏感字段）
- 产出：指标汇总、快照列表

### SC-23 消息通知与未读数

- 分类：CAT-E（指标与通知）
- 角色：全角色
- 场景：
  - 分配专家/任务状态变化/报告生成等触发通知
  - 查看未读数、批量已读、清理已读
- 产出：通知状态变更

### SC-24 权限不足/接口保护失败

- 分类：CAT-F（异常与运维排障）
- 角色：全角色
- 常见情况：
  - 未登录访问受限接口 → 401
  - 非授权角色访问 → 403
  - 管理写接口未配置 `ADMIN_API_KEY` 且非管理员token访问 → 500（配置缺失）
- 期望体验：
  - 前端清晰提示并引导登录/联系管理员
  - 后端返回结构化错误信息，便于排障

### SC-25 环境与端口冲突（部署侧）

- 分类：CAT-F（异常与运维排障）
- 角色：运维/部署人员
- 常见问题：
  - 宿主机 80/443/8000 端口被占用导致容器启动失败或无法访问
  - Nginx 反代未指向后端容器内端口导致 `/api` 502
- 期望：
  - 统一对外仅暴露 80，后端在容器内 8000
  - 部署手册提供可检查项：容器状态、日志、健康检查

### SC-26 系统清单维护与批量导入（含模板下载）

- 分类：CAT-G（配置管理（基础））
- 角色：admin
- 目标：维护主系统清单与子系统映射，为系统识别提供“权威名称与映射”基础数据。
- 主流程：
  1. 进入系统清单配置页面
  2. 下载模板（XLSX）
  3. 选择批量导入文件进行预览（尽量容错：允许重复/空值等）
  4. 确认导入（追加/覆盖按页面策略）
  5. 在页面继续编辑单条系统/映射（支持重复名称时以稳定ID为准）
- 产出：主系统清单、子系统映射、导入预览与确认结果
- 异常/边界：
  - 模板表头无法识别/缺Sheet → 导入失败并给出明确提示
  - XLSX兼容性（含校验规则等） → 系统应做兼容处理或给出可执行的修复建议

### SC-27 COSMIC规则配置（规则管理）

- 分类：CAT-G（配置管理（基础））
- 角色：admin
- 目标：维护 COSMIC 数据移动判定规则，供分析与展示使用。
- 主流程：
  1. 打开 COSMIC 配置页面
  2. 调整关键词/权重/粒度等配置并保存
  3.（如有）触发 reload/生效操作
- 产出：COSMIC配置、（可选）生效日志
- 异常/边界：
  - 配置不合法（空规则/权重异常） → 阻止保存或提示修复

### SC-28 用户管理与批量导入（含专家库字段）

- 分类：CAT-G（配置管理（基础））
- 角色：admin
- 主流程：
  1. 下载用户导入模板（XLSX）
  2. 上传文件进行导入预览（校验用户名/角色等）
  3. 确认导入，写入用户数据
  4. 对用户进行启用/禁用、角色调整、密码重置（如页面提供）
  5. 维护专家库相关字段：部门（下拉）、专长（多标签）、在岗/休假
- 产出：用户/专家库数据、导入结果
- 异常/边界：
  - 管理写接口保护：需 admin 权限或 ADMIN_API_KEY
  - 专家在岗校验：分配专家时会阻断“未在岗/非expert/已禁用”账号

### SC-29 系统知识库管理（system_profile）导入与检索

- 分类：CAT-C（知识/证据/索引）
- 角色：manager
- 目标：导入与检索系统知识（当前仅 system_profile 口径），用于系统识别与拆分的约束/提示信号。
- 主流程：
  1. 在知识库管理页面选择主系统（system_name）
  2. 导入 DOCX/PPTX，系统自动抽取结构化信息并写入向量库
  3. 输入查询文本进行检索（按 system_name 过滤）
  4. 查看统计与效果指标（命中率/相似度等）
  5. 必要时重建索引
- 产出：系统知识条目、检索结果、统计与指标
- 异常/边界：
  - 未选择 system_name → 阻止导入/检索
  - 缺少模型 Key 导致 embedding 不可用 → 返回可理解错误或走降级策略（以部署配置为准）
