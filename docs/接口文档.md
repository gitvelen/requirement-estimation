## 1. 概述
- **Base URL**：`http(s)://<host>:<port>/api/v1`
- **OpenAPI（如启用）**：`/docs`（Swagger UI）、`/openapi.json`
- **版本策略**：当前主版本为 `/api/v1`；v2.0/v2.1 新增接口统一采用“错误码（error_code）+ request_id”结构返回错误（见 2.2）

### 1.1 认证与授权
- **JWT 登录**：
  - `POST /api/v1/auth/login` 获取 token
  - 后续请求携带：`Authorization: Bearer <token>`
- **管理员导入类接口（可选叠加保护）**：
  - 部分 admin 写接口支持 `ADMIN_API_KEY`
  - 推荐请求头：`X-API-Key: <ADMIN_API_KEY>`

### 1.2 幂等与重试（摘要）
- 幂等接口（典型）：代码扫描任务提交/入库、导入类接口、看板查询、任务查询等（以 `docs/v2.0/requirements.md` 的 API-001~018 约定为准）
- 建议客户端重试策略：
  - 网络失败/超时：可重试幂等接口
  - 非幂等接口：避免自动重试，或在业务侧引入幂等键

## 2. 通用约定

### 2.1 请求头/追踪
| Header | 说明 |
|---|---|
| `Authorization` | `Bearer <JWT>`（多数接口） |
| `X-API-Key` | 管理员导入类接口可选叠加保护 |
| `X-Request-ID` | 可选：请求追踪 ID；v2.0 错误响应会返回 `request_id`（缺省由服务端生成） |

### 2.2 返回格式与错误码
**成功响应（常见）**：
```json
{ "code": 200, "message": "success", "data": {} }
```
或：
```json
{ "code": 200, "data": {} }
```

**v2.0 错误响应（新接口）**：
```json
{
  "error_code": "AUTH_001",
  "message": "权限不足",
  "request_id": "req_20260208120000_1a2b3c4d",
  "details": { "reason": "..." }
}
```

> 说明：部分历史接口仍可能返回 FastAPI 默认的 `{"detail": "..."}` 或旧 `{code,message}` 结构；客户端需兼容处理。

**错误码清单**：以 `docs/v2.0/requirements.md` 的“6.2 错误码与提示语”为准（本文档不重复枚举全部错误码）。

## 3. 接口列表（截面式）
> 本节只列“当前系统入口”，详细字段/约束以 `docs/v2.0/requirements.md` 的“6.4 API 需求”与 OpenAPI 为准。

### 3.1 认证
- `POST /auth/login`：登录获取 JWT
- `GET /auth/me`：获取当前用户
- `POST /auth/change-password`：修改密码

### 3.2 健康检查
- `GET /health`：健康检查（期望 `healthy`）

### 3.3 任务与评估（主链路）
- **API-016** `POST /tasks`：创建评估任务（`multipart/form-data`，支持 `.docx/.doc/.xls`，默认≤10MB）
- **API-010** `GET /tasks`：任务查询（filters + group_by_status + 权限范围）
- `GET /tasks/{task_id}`：任务详情（admin 全量；manager 创建者；expert 参与任务可读）
- `GET /tasks/{task_id}/high-deviation`：高偏离功能点（参与任务 expert 可读）
- **API-013** `GET /tasks/{task_id}/evaluation`：评估详情契约
- **API-007** `POST /tasks/{task_id}/modification-traces`：修改轨迹记录
- `GET /tasks/{task_id}/expert-deviations`：专家差异统计查询（见 API-008 相关）
- **API-014** `GET /tasks/{task_id}/report`：报告下载（v2.0 仅 PDF；docx 预留返回错误码）

### 3.4 系统画像与知识导入
- **API-001** `POST /code-scan/jobs`、`GET /code-scan/jobs/{job_id}`：代码扫描任务提交/查询
- **API-002** `POST /code-scan/jobs/{job_id}/ingest`：扫描结果入库（更新完整度）
- **API-003** `POST /esb/imports`：ESB 文档导入（可选 `mapping_json`，支持 `.xls` 历史格式）
- **API-011** `POST /knowledge/imports`：知识导入（document/code；支持 normal/L0，文档入口支持 `.doc/.docx/.xls`）
- **API-004** `/system-profiles...`：系统画像（列表/草稿/发布/只读）
- **API-012** `GET /system-profiles/completeness`：材料完整度
- **API-018** `POST /system-profiles/{system_id}/ai-suggestions/retry`：画像 AI 总结重试（主责/B角）
- **API-005（internal）** `POST /internal/system-profiles/retrieve`：内部检索聚合
- **API-006（internal）** `POST /internal/complexity/evaluate`：复杂度三维度评估

### 3.5 效能看板
- **API-009** `POST /efficiency/dashboard/query`：看板查询（widgets + drilldown_filters）

### 3.6 系统清单（管理员）
- **API-015** `GET /system-list/template`：下载导入模板
- **API-015** `POST /system-list/batch-import`：批量导入预览校验（不落盘）
- **API-015** `POST /system-list/batch-import/confirm`：确认导入并热加载

### 3.7 通知中心与画像AI建议
- **API-017** `GET /notifications`：通知列表（`items` + 兼容 `data` 字段）
- **API-017** `GET /notifications/unread-count`：未读计数（`unread_count`）
- **API-017** `PUT /notifications/{notification_id}/read`、`PUT /notifications/read-all`：标记已读
- **API-017** `DELETE /notifications/clear-read`、`DELETE /notifications/{notification_id}`：清理已读/删除单条
- 通知类型新增：`system_profile_summary_ready`、`system_profile_summary_failed`

### 3.8 v2.1 增量接口（重点）
- **API-007** `GET /system/config/feature-flags`：查询 `V21_AUTO_REEVAL_ENABLED`、`V21_AI_REMARK_ENABLED`、`V21_DASHBOARD_MGMT_ENABLED`。
- **API-001** `PUT /requirement/features/{task_id}`：仅落库并写修改记录（补充 `actor_id/actor_role`），客户端 `remark` 字段忽略。
- **API-006** `POST /tasks/{task_id}/reevaluate`：独立触发重评估（同 task 幂等，重复请求返回已有 job 状态）。
- **API-002/API-003** `PUT/GET /system-profiles/{system_name}`：画像字段收敛为 4 项（`system_scope`、`module_structure`、`integration_points`、`key_constraints`）；`module_structure` 非法返回 `invalid_module_structure`。
- **API-004** `POST /efficiency/dashboard/query`：管理驱动指标扩展（6 项指标 + AI 学习趋势）；`filters.ai_involved` 在 v2.1 兼容忽略。

## 4. 废弃/兼容接口（摘要）
| 类型 | 路径 | 状态 | 替代方案 |
|---|---|---|---|
| 兼容 | `/api/v1/code-scan/run` / `/status` / `/result` / `/commit` | 保留兼容入口（不推荐新接入） | `POST /api/v1/code-scan/jobs` + `POST /ingest` |
| 兼容 | `/api/v1/knowledge/import`（旧） | 保留兼容（不推荐新接入） | `POST /api/v1/knowledge/imports` |
| 前端兼容 | `/reports/ai-effect` | 入口移除，自动跳转 | `/dashboard`（效能看板） |
