## 1. 变更意图（User Story 格式）
- **作为** 项目经理
- **我想要** 信息展示页在画像保存了旧 `system_id` 时仍能正常打开并加载时间线/建议操作
- **以便于** 系统清单 ID 变化后页面不再报“系统不存在”

## 2. 变更点（What）
- 信息展示页新增统一 `effectiveSystemId`，优先使用系统清单当前系统 ID，回退到画像 `system_id`。
- `loadProfileDetail` 中时间线请求 ID 解析改为“系统清单 ID 优先”。
- 保存草稿、发布画像、采纳建议、回滚建议、加载更多时间线统一使用 `effectiveSystemId`。
- 新增前端回归断言：当画像携带过期 ID 时，请求必须使用当前系统清单 ID。
- 后端事件查询容错修复：当系统已存在但画像尚未创建时，`/profile/events` 返回空列表（`total=0, items=[]`），不再错误返回 404。
