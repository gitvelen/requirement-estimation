## 1. 系统概述

### 1.1 面向用户与范围

**面向用户/主要使用者**
- admin（管理员）：系统治理与配置（用户/系统清单/证据等级规则等）、查看全部任务、分配专家、审计追踪
- manager（项目经理）：发起任务、复核系统边界与功能点、提交任务进入评估、维护证据/画像/ESB/代码扫描等事实信号、查看报告与效果指标
- expert（专家）：参与评估（多轮盲评）、提交/撤回评估、在线预览任务关联证据（不提供下载）
- 外部系统（可选集成）：DevOps/评审系统通过 API 触发评估并接收回调

**适用范围（In Scope）**
- 角色化入口与权限边界：admin/manager/expert
- 任务主链路：上传需求→AI处理→人机协作修正→提交/分配→多轮专家评估→报告版本→归档
- 采购严肃增强（可审计/可追溯）：
  - A层证据库：材料导入、切块检索、在线预览（不提供下载）
  - B层系统画像：草稿/发布、字段可回链证据
  - I层 ESB 索引：明细导入/检索/统计（仅集成点提示，不扩范围）
  - C层 代码扫描：Spring Boot/Java 轻量扫描产出能力目录（事实层，不扩范围）
  - 证据等级（E0~E3）规则配置与任务内覆盖修正留痕
  - 区间估算 + 置信度 + 假设 + 风险提示
- 报告与指标：
  - 多轮评估报告（PDF）与版本管理
  - 排行榜与多维报表（v2.2 固定近90天口径）
  - AI效果报告入口已下线（保留旧 URL 兼容跳转提示）
- 基础配置管理：系统清单、COSMIC规则、用户/专家库字段、部门字典、通知中心、个人中心
- 对外集成：RESTful API；支持 DevOps/外部系统调用评估接口（可选回调）

**不包含范围（Out of Scope）**
- 端到端自动报价/自动招采定价（仅提供评估与风控提示）
- 全量代码语义理解与自动扩范围（任何召回信号仅用于“提示复核”，不得自动扩大需求范围）
- 扫描版 PDF OCR（本期仅保证可抽取文本的 PDF）
- 生产级数据库/缓存/对象存储/K8s/CI-CD 的完整落地（当前以文件存储为主，可平滑演进）

### 1.2 术语与口径

**系统识别分层**
- `selected_systems`：当前证据足以“确定涉及”的系统（默认仅 E2/E3 进入）
- `maybe_systems`：证据不足或存在冲突的候选系统
- `questions`：最小澄清问题集（默认 3~8 条；按优先级截断）

**证据等级（E0~E3，默认规则可配置）**
- E3：具备“代码扫描 +（证据或画像）”等强证据组合
- E2：具备“证据/画像”或“代码/ESB”等中等证据组合；允许进入 selected
- E1：证据较弱（仅证据/画像/ESB 之一）；默认进入 maybe
- E0：无可用证据；默认进入 maybe 并强提示高风险

**偏离度口径**
- 偏离度 = `|专家均值 - AI预估| / AI预估 × 100%`（AI预估为 0 时偏离度记为 0）
- 高偏离阈值：默认 20%，用于触发下一轮评估与高偏离列表

**编号规则**
- 场景：`SCN-001` 起
- 功能：`FUNC-001` 起
- 接口：`API-001` 起（与 `docs/接口文档.md` 对齐）

**ID 前缀规则**：详见 `.aicoding/STRUCTURE.md` 统一定义。

### 1.7 v2.4 增量能力

- 文档导入页改为文档类型卡片化独立操作区，支持类型内状态隔离、导入历史列表与“查看系统画像”直达。
- 系统画像从 4 扁平字段升级为 5 域 12 子字段结构，信息展示页采用“域导航 + 内容区 + 时间线”三区布局，并支持子字段级 inline diff 与 AI 建议采纳/忽略/回滚。
- 文档导入与代码扫描入库后异步触发 AI 结构化提取，按域相关性更新 `ai_suggestions`，并保留 `ai_suggestions_previous` 用于一级回滚。
- 工作量估算由静态映射切换为 LLM 三点估计（`optimistic/most_likely/pessimistic/expected/reasoning`），评估页与报告页可展开查看明细。
- 评估流水线新增 `ai_original_output` 快照、`pm_correction_diff`、`ai_correction_history`，形成 AI 输出与 PM 修正的可追溯沉淀链。


## 2. 业务场景说明（必须覆盖所有场景）

### 2.1 角色与对象

**角色**
- 管理员（admin）
- 项目经理（manager）
- 专家（expert）
- 外部系统：DevOps/评审系统（可选集成方）

**核心对象**
- 任务 Task：一次评估的载体（需求正文、系统识别分层、拆分结果、评估轮次、报告版本、审计留痕）
- 功能点 Feature：按系统拆分的最小可估算单元（输入/输出/依赖、功能类型、区间估算、置信度、假设、风险）
- 证据材料 EvidenceDoc：DOCX/PPTX/PDF 原件 + 切块索引（A层）
- 系统画像 SystemProfile：结构化边界摘要（B层）
- ESB 条目 EsbEntry：集成点明细（I层）
- 代码扫描作业 CodeScanJob/结果：能力目录（C层）
- 报告版本 ReportVersion：每轮生成的 PDF（以及可选的 Excel 产物）
- 通知 Notification：站内通知（未读数、已读状态）

### 2.3 场景明细 [包含2.2中所有场景]

#### SCN-001：首次访问与登录
**场景分类**：CAT-A 账号与权限
**主要角色**：admin/manager/expert
**相关对象**：User、JWT
**关联功能项ID**：FUNC-001
**前置条件**：
- 用户账号已创建且启用
**触发条件**：
- 用户访问系统并提交用户名密码
**流程步骤**：
1. 用户输入用户名密码并登录
2. 系统签发 JWT 并返回角色信息
3. 前端保存 token 并按角色跳转首页
**输出产物**：
- access_token、user基本信息
**异常与边界处理**：
- 密码错误返回 401；禁用账号禁止登录

#### SCN-002：账号治理与安全
**场景分类**：CAT-A 账号与权限
**主要角色**：admin/全角色
**相关对象**：User、Role、Profile
**关联功能项ID**：FUNC-002、FUNC-028、FUNC-030
**前置条件**：
- admin 已登录（用户管理）；或用户已登录（个人中心）
**触发条件**：
- admin 管理用户/角色/状态；用户改密或维护资料
**流程步骤**：
1. admin 在用户管理中增删改查用户、启停用、维护角色与专家字段
2. 用户在个人中心修改密码/头像/资料
3. 系统记录关键操作日志（如适用）
**输出产物**：
- 用户状态/角色变更；活动日志
**异常与边界处理**：
- 非 admin 访问用户管理返回 403；改密校验旧密码

#### SCN-003：创建任务（上传需求）
**场景分类**：CAT-B 任务评估主链路
**主要角色**：manager
**相关对象**：Task、RequirementDoc
**关联功能项ID**：FUNC-003
**前置条件**：
- manager 已登录；DOCX 文件合法且≤10MB
**触发条件**：
- 项目经理上传需求文档并发起评估
**流程步骤**：
1. 上传 DOCX 创建任务
2. 后端解析需求正文并触发 AI 编排
3. AI完成后任务进入草稿，进入编辑页复核
**输出产物**：
- task_id、aiStatus/progress
**异常与边界处理**：
- 非 DOCX 返回 400；文件过大返回 413

#### SCN-004：AI失败与降级
**场景分类**：CAT-F 异常与运维排障
**主要角色**：manager/admin
**相关对象**：Task、AIJob
**关联功能项ID**：FUNC-004、FUNC-024
**前置条件**：
- 已创建任务并触发 AI
**触发条件**：
- 模型 Key 缺失、网络异常、模型服务故障等导致 AI 失败
**流程步骤**：
1. AI失败写入任务 ai_status=failed 与 error_message
2. 前端展示原因并提供重试/人工链路兜底
3. 运维可通过日志与健康检查定位问题
**输出产物**：
- 可理解错误提示；可回放日志
**异常与边界处理**：
- 错误信息不泄露敏感信息；保留任务数据以便追溯

#### SCN-005：系统识别复核
**场景分类**：CAT-B 任务评估主链路

- 外部系统通过 API 上传需求触发评估
**流程步骤**：
1. 外部系统上传 DOCX 与 request_id、callback_url（可选）
2. 系统创建任务并后台处理
3. 处理结束回调 callback_url（成功/失败均回调）
**输出产物**：
- task_id/status；回调 payload
**异常与边界处理**：
- callback_url 不满足策略返回 400（防 SSRF）

### 3.2 功能项明细 [包含3.1中所有功能项]

#### FUNC-001：登录与会话
**目标/价值**：建立安全的登录态（JWT），为后续角色化功能提供统一入口。  
**入口/触发**：登录页；`POST /api/v1/auth/login`、`GET /api/v1/auth/me`。  
**前置条件**：用户账号存在且启用。  
**主流程**：
1. 用户提交用户名密码
2. 后端校验通过后签发 JWT，并返回用户基本信息与 roles
3. 前端保存 token，并按角色跳转首页
**输入/输出**：
- 输入：username、password
- 输出：token、user（id/username/displayName/roles/avatar）
**业务规则**：
- 禁用账号禁止登录
**异常与边界**：
- 用户名/密码错误返回 401
**验收标准（GWT，必须可测试）**：
- [ ] Given 用户名密码正确且账号启用，When 调用登录接口，Then 返回 JWT 且包含 roles
- [ ] Given 用户名或密码错误，When 调用登录接口，Then 返回 401 且提示“用户名或密码错误”
- [ ] Given 未携带 JWT，When 调用受保护接口，Then 返回 401 并跳转登录页
**关联**：SCN-001

#### FUNC-002：账号治理与安全
**目标/价值**：支持账号生命周期管理与基础安全能力（改密/启停用/角色）。  
**入口/触发**：用户管理页（admin）；个人中心（全角色）。  
**前置条件**：admin 已登录；或用户已登录。  
**主流程**：
1. admin 管理用户：增删改查、启用/禁用、分配角色
2. 用户自助：修改密码、维护基础资料与头像
**输入/输出**：
- 输入：用户信息、角色、状态；oldPassword/newPassword；头像文件
- 输出：用户变更结果、活动日志（如适用）
**业务规则**：
- admin 不可被禁用导致系统无法治理（建议运维约束；实现可通过流程控制）
**异常与边界**：
- 非 admin 访问用户管理接口返回 403/401
**验收标准（GWT，必须可测试）**：
- [ ] Given admin 已登录，When 禁用某用户，Then 该用户后续登录返回 403
- [ ] Given 用户已登录，When 修改密码成功，Then 旧密码不可再登录且新密码可登录
**关联**：SCN-002、SCN-028、SCN-030

#### FUNC-003：创建任务（上传需求）
**目标/价值**：将 DOCX 需求文档转为可编辑的评估任务（草稿）。  
**入口/触发**：上传需求页；`POST /api/v1/tasks`。  
**前置条件**：manager 已登录；文件为 DOCX 且≤10MB。  
**主流程**：
1. 上传 DOCX 创建任务
2. 后台解析需求正文并触发 AI 编排处理
3. 任务完成后进入草稿，可在编辑页复核
**输入/输出**：
- 输入：DOCX 文件、可选任务名称/说明
- 输出：taskId、aiStatus/progress/message
**业务规则**：
- 仅 manager 可创建任务
**异常与边界**：
- 文件格式非 DOCX 返回 400；文件过大返回 413
**验收标准（GWT，必须可测试）**：
- [ ] Given manager 已登录，When 上传合法 DOCX，Then 返回 taskId 且任务可在“我发起的任务”看到
- [ ] Given 非 manager 用户，When 调用创建任务接口，Then 返回 403
**关联**：SCN-003

#### FUNC-004：AI失败与降级
**目标/价值**：AI 失败时不“黑盒”，给出可理解原因并提供可继续推进路径。  
**入口/触发**：任务列表/进度接口；AI 处理后台。  
**前置条件**：任务已创建。  
**主流程**：
1. AI 失败时写入任务 error/message
2. 前端展示错误并允许用户重试/人工处理（产品策略）
**输入/输出**：
- 输入：任务ID
- 输出：aiStatus=failed、error/message
**业务规则**：
- 错误信息应可理解且不泄露敏感信息
**异常与边界**：
- 未配置模型 Key 时需明确提示
**验收标准（GWT，必须可测试）**：
- [ ] Given 模型 Key 缺失，When 发起 AI 任务，Then 任务进入 failed 且 message 提示需配置 Key
**关联**：SCN-004

#### FUNC-005：系统识别分层复核
**目标/价值**：以“证据约束 + 阻断”提升系统识别准确性与可审计性。  
**入口/触发**：编辑页系统校准卡片；任务结果接口。  
**前置条件**：任务 AI 初评完成（草稿）。  
**主流程**：
1. 输出 selected/maybe/questions
2. selected 默认仅包含证据等级 E2/E3 系统；其余进入 maybe
3. questions 控制在 3~8 条，面向业务可回答
**输入/输出**：
- 输入：任务ID、需求正文、证据/画像/ESB/扫描等索引信号
- 输出：ai_system_analysis（分层+证据引用+风险提示）
**业务规则**：
- 严肃模式下：证据不足禁止输出确定结论
**异常与边界**：
- 未命中证据时需提示缺失项（material/profile/code/esb）
**验收标准（GWT，必须可测试）**：
- [ ] Given 系统证据等级为 E1/E0，When 输出系统识别结果，Then 该系统进入 maybe_systems 且 questions 提示补充证据
- [ ] Given questions 超过 8 条，When 返回结果，Then 按优先级截断并返回前 3~8 条
**关联**：SCN-005

#### FUNC-006：新增系统Tab自动拆分
**目标/价值**：支持补齐系统范围并自动生成该系统功能点初稿。  
**入口/触发**：编辑页“新增系统Tab”；`POST /api/v1/requirement/systems/{task_id}`。  
**前置条件**：任务草稿可编辑且存在需求正文。  
**主流程**：
1. 新增系统Tab
2. 默认触发自动拆分；失败保留空Tab
**输入/输出**：
- 输入：task_id、system_name、（可选）system_type、auto_breakdown
- 输出：新增系统与拆分结果（或错误提示）
**业务规则**：
- 新增系统后默认进入可编辑草稿
**异常与边界**：
- 需求正文缺失时禁止自动拆分并提示
**验收标准（GWT，必须可测试）**：
- [ ] Given 任务草稿且需求正文存在，When 新增系统Tab，Then 生成该系统功能点列表（或返回空Tab并提示失败原因）
**关联**：SCN-006

#### FUNC-007：单系统重拆（rebreakdown）
**目标/价值**：允许对单系统拆分质量进行定向修复，避免全量重跑。  
**入口/触发**：编辑页“重新拆分”；`POST /api/v1/requirement/systems/{task_id}/{system_name}/rebreakdown`。  
**前置条件**：任务草稿可编辑。  
**主流程**：
1. 触发重拆并覆盖该系统功能点
2. 失败则保留原结果并提示原因
3. 写入修改记录
**输入/输出**：
- 输入：task_id、system_name
- 输出：覆盖后的功能点、breakdown_error（如有）
**业务规则**：
- 仅覆盖当前系统，不影响其他系统
**异常与边界**：
- 重拆失败不应破坏原数据
**验收标准（GWT，必须可测试）**：
- [ ] Given 重拆失败，When 返回结果，Then 原功能点列表保持不变且返回 breakdown_error
**关联**：SCN-007

#### FUNC-008：功能点编辑与审计
**目标/价值**：支持人机协作修正，形成可回放修改记录与后续校准数据。  
**入口/触发**：编辑页；`PUT /api/v1/requirement/features/{task_id}` 等系统操作接口。  
**前置条件**：任务处于 draft 且未提交。  
**主流程**：
1. 功能点增删改（模块/名称/描述/输入/输出/依赖等）
2. 系统Tab：重命名、删除、新增、重拆
3. 每次修改写入 modifications（用于报告与指标）
**输入/输出**：
- 输入：操作类型（add/update/delete）、feature_data
- 输出：更新成功与最新 modifications
**业务规则**：
- 已确认/已提交/已归档任务不可修改
**异常与边界**：
- 非法索引/系统不存在返回 400
**验收标准（GWT，必须可测试）**：
- [ ] Given 任务为草稿，When 修改功能点业务描述，Then modifications 记录包含 operation=update 且可查询
- [ ] Given 任务已提交，When 再次修改功能点，Then 返回 400 且提示“该任务已确认/不能修改”
**关联**：SCN-008

#### FUNC-009：提交给管理员分配专家
**目标/价值**：将草稿冻结进入评估流程，确保评估口径一致。  
**入口/触发**：编辑页“提交给管理员”；`POST /api/v1/tasks/{task_id}/submit-to-admin`。  
**前置条件**：任务处于 draft，manager 对任务有访问权限。  
**主流程**：
1. draft → awaiting_assignment
2. 发送通知给 admin
**输入/输出**：
- 输入：task_id
- 输出：提交成功
**业务规则**：
- 非草稿状态不可提交
**异常与边界**：
- 任务不存在返回 404
**验收标准（GWT，必须可测试）**：
- [ ] Given 任务为 draft，When 提交给管理员，Then 状态变为 awaiting_assignment 且 admin 收到通知
- [ ] Given 任务非 draft，When 提交给管理员，Then 返回 400 且提示“当前状态不可提交”
**关联**：SCN-009

#### FUNC-010：分配专家与校验
**目标/价值**：确保评估由合格专家完成并进入可控多轮流程。  
**入口/触发**：任务管理“分配专家”；`POST /api/v1/tasks/{task_id}/assign-experts`。  
**前置条件**：任务为 awaiting_assignment；存在至少3位在岗专家。  
**主流程**：
1. admin 选择 3 位专家并提交
2. 后端校验专家账号与在岗状态
3. 生成邀请记录/通知，任务进入 evaluating（round=1）
**输入/输出**：
- 输入：task_id、expert_ids[3]
- 输出：分配结果、assignment记录
**业务规则**：
- 必须分配 3 位专家
- 专家必须是系统用户且角色为 expert、账号启用、在岗
**异常与边界**：
- 专家不满足条件返回 400 并说明原因
**验收标准（GWT，必须可测试）**：
- [ ] Given admin 已登录且任务待分配，When 分配 3 位在岗专家，Then 任务进入 evaluating 且专家收到通知
- [ ] Given 专家 on_duty=false，When 试图分配该专家，Then 返回 400 且提示“专家不在岗”
**关联**：SCN-010

#### FUNC-011：专家评估（草稿/提交/撤回）
**目标/价值**：支持专家独立评估并形成可回放结果。  
**入口/触发**：评估页；`GET/POST /api/v1/evaluation/*`。  
**前置条件**：专家已被分配到任务轮次。  
**主流程**：
1. 拉取当前轮次需评估功能点（第1轮=全部；第2/3轮=高偏离）
2. 专家修改人天并自动保存草稿
3. 提交评估；在报告生成前可撤回（受限）
**输入/输出**：
- 输入：feature_id → days；提交/撤回动作
- 输出：草稿/提交状态；时间戳
**业务规则**：
- 专家只能评估自己被分配的任务与轮次
- 撤回仅在未生成报告前允许
**异常与边界**：
- 越权访问返回 403
**验收标准（GWT，必须可测试）**：
- [ ] Given expert 被分配任务，When 修改某功能点人天，Then 自动保存草稿且刷新不丢失
- [ ] Given expert 已提交且报告未生成，When 撤回，Then 状态变为 revoked 并允许重新提交
**关联**：SCN-011

#### FUNC-012：多轮评估触发
**目标/价值**：通过多轮盲评降低偏差并收敛争议点。  
**入口/触发**：评估提交后自动触发；任务详情页查看。  
**前置条件**：当前轮次3位专家均已提交。  
**主流程**：
1. 计算每个功能点专家均值与偏离度
2. 若存在偏离度>阈值功能点，进入下一轮（最多3轮）
3. 下一轮仅评估高偏离功能点
**输入/输出**：
- 输入：本轮 evaluations
- 输出：round_means、deviations、high_deviation_features、next_round
**业务规则**：
- 阈值默认 20%（可配置）
- 达到 max_rounds 后不再触发下一轮
**异常与边界**：
- AI预估为 0 的偏离度口径按 0 处理（避免除零）
**验收标准（GWT，必须可测试）**：
- [ ] Given 偏离度>20%，When 完成第1轮提交，Then 系统进入第2轮且仅返回高偏离功能点
- [ ] Given 偏离度均≤20%，When 完成第1轮提交，Then 直接生成报告并标记 completed
**关联**：SCN-012

#### FUNC-013：报告生成与版本管理
**目标/价值**：将评估过程沉淀为可交付、可追溯的报告版本。  
**入口/触发**：每轮评估完成自动生成；下载接口。  
**前置条件**：当前轮次评估完成。  
**主流程**：
1. 汇总系统/功能点/区间/均值/偏离度/风险/证据等级等
2. 生成 PDF 报告并记录 report_versions
3. 支持下载历史版本
**输入/输出**：
- 输入：task_id、round
- 输出：report_id、path、created_at
**业务规则**：
- 每轮生成一个版本；版本不可篡改（至少文件名/路径稳定）
**异常与边界**：
- 生成失败需返回原因并保留任务数据
**验收标准（GWT，必须可测试）**：
- [ ] Given 第1轮评估完成，When 生成报告，Then report_versions 新增一条且可下载 PDF
**关联**：SCN-013

#### FUNC-014：归档/删除/保留策略
**目标/价值**：降低任务列表噪声并控制存储成本，同时保证审计可追溯。

#### FUNC-015：证据库（A层）
**目标/价值**：提供可回放的事实材料，支撑可审计输出与准确性提升。  
**入口/触发**：证据库页面；证据导入/检索/统计接口。  
**前置条件**：manager/admin 已登录；文件格式合法。  
**主流程**：

**输入/输出**：

**验收标准（GWT，必须可测试）**：

## 5. 关键流程说明

- 流程1：需求评估主流程（manager→admin→experts→报告）
- 流程2：证据预览权限控制（expert范围收敛）
- 流程3：严肃模式阻断（证据等级 gating）
- 流程4：DevOps 外部调用与回调（防 SSRF）

### 5.1 需求评估主流程（简化）

```
上传DOCX → 文档解析 → 系统识别 → 功能拆分 → 区间估算
    ↓
项目经理复核/修正 → 提交给管理员 → 分配3位专家
    ↓
第1轮盲评（全部功能点）→ 计算偏离度 → （需要则）第2/3轮（仅高偏离）
    ↓
生成报告版本（PDF）→ 下载/追溯 → 归档/保留策略
```

### 5.2 证据预览权限控制（expert）

```
expert 预览请求(doc_id, task_id)
  → 校验 expert 是否参与该 task_id
  → 校验 doc_id 是否属于该任务关联证据
  → 返回 inline 预览（不提供下载）
```

### 5.3 严肃模式阻断（证据等级）

```
计算 evidence_level(E0~E3)
  → E2/E3：允许进入 selected_systems
  → E1/E0：默认进入 maybe_systems，并输出 questions + 高风险提示
```

### 5.4 DevOps外部调用（带回调）

```
外部系统 POST /api/v1/requirement/evaluate (request_id, callback_url, file)
  → 校验 callback_url 为内网/本机（防SSRF）
  → 创建任务并后台处理
  → 回调 callback_url（成功/失败均回调）
```

## 6. 权限与角色（面向使用者）

| 角色 | 可见功能 | 关键限制 |
|---|---|---|
| admin | 全局任务视图、分配专家、系统清单、用户/专家库、证据等级规则与审计、查看报告与效果 | 不可兼职其他角色（产品约束）；管理写接口需管理员鉴权或二级保护 |
| manager | 上传需求、复核/编辑草稿、提交任务、维护证据/画像/ESB/扫描、查看报告与指标 | 默认仅可访问“我发起的任务”；可兼任 expert；任务内覆盖证据等级需备注留痕 |
| expert | 查看“我参与的评估”、多轮评估、草稿/提交/撤回、在线预览任务关联证据、查看匿名效果指标 | 仅可访问参与任务范围；证据仅在线预览不下载；第二/三轮仅评估高偏离功能点 |
| 外部系统（可选） | DevOps评估接口触发任务、可选回调接收结果 | callback_url 必须通过内网/本机校验；回调失败重试由集成方策略处理 |

## 7. 常见问题与错误处理

### 7.1 常见问题（示例）

| 问题 | 可能原因 | 处理建议 |
|---|---|---|
| 上传需求失败 | 文件非 DOCX 或超过大小限制 | 确认格式与大小（默认≤10MB） |
| AI处理失败/长时间无响应 | 未配置模型Key/网络异常/模型服务故障 | 检查 `DASHSCOPE_API_KEY`、查看后端日志，必要时走人工链路 |
| 无法分配专家 | 专家未启用/不在岗/角色不匹配 | 在用户管理中检查 `roles/is_active/on_duty` |
| 无法预览证据 | 未参与任务或证据未关联任务 | 确认参与关系与任务关联证据清单 |
| 报告生成失败 | 任务数据不完整或异常中断 | 查看任务状态与日志，补齐必填字段后重试 |
| `/api` 502 | Nginx反代或后端容器异常 | 检查 `frontend/nginx.conf` 与 `docker-compose` 容器网络/日志 |

### 7.2 常见错误码与提示语（示例）

| 错误码 | 场景 | 提示语 | HTTP状态码 | 处理建议 |
|---|---|---|---|---|
| AUTH_MISSING | 未登录访问受保护接口 | 缺少Authorization | 401 | 重新登录 |
| AUTH_FORBIDDEN | 角色不匹配/越权访问 | 权限不足 | 403 | 切换账号/联系管理员 |
| ADMIN_KEY_MISSING | 管理接口缺少配置 | ADMIN_API_KEY 未配置 | 500 | 配置 `ADMIN_API_KEY` 或使用 admin JWT |
| FILE_TOO_LARGE | 上传文件过大 | 文件过大，最大允许XXMB | 413 | 压缩/拆分文件 |
| INVALID_FILE_TYPE | 文件类型不支持 | 仅支持 DOCX/PPTX/PDF | 400 | 更换文件格式 |
| CALLBACK_URL_INVALID | 外部回调地址不合法 | 回调地址必须为内网或本机地址 | 400 | 使用内网地址或取消回调 |
