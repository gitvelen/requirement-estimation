## 1. 概述

### 1.1 目的与范围
本文档将 `docs/v2.2/proposal.md` 中的 v2.2 提案落为**可验收、可追溯**的需求（REQ/REQ-C/GWT），作为后续 Design/Planning/Implementation/Testing 的单一需求输入。

**In Scope（与 proposal.md 一致）**：
- 全站页面标题精简（删除冗余 PageHeader）
- 效能看板导航重构：排行榜/多维报表子页面；兼容旧 URL；下线 AI表现/FUNC-022
- 任务管理子菜单化（进行中/已完成）+ 旧 URL 兼容
- 任务详情页摘要优化；移除“报告版本列表”区域但保留历史报告下载入口
- 项目经理编辑管控：预估人天只读；实质性修改“保存并确认”触发 AI 重评估
- 系统画像“知识导入”页：代码扫描/文档导入 Tab；文档导入支持 5 种类型；ESB 特有能力保留
- 专家评估页：COSMIC 规则入口从右侧大卡片改为顶部“?”图标
- 备注字段：AI 自动生成只读展示（保留历史人工备注）
- Deployment 阶段同步更新主文档（系统功能说明书/接口文档/用户手册）

**Out of Scope / Non-goals**（与 proposal.md 一致）：
- 数据库迁移、数据格式/存储结构变更
- 新增角色或重做权限体系
- 移动端适配、国际化（i18n）
- 性能专项优化（非本次目标）
- FUNC-022（AI效果报告）的保留/迁移（v2.2 随“AI表现”模块一并下线）

### 1.2 背景、约束与关键假设
**现状与痛点**：见 `docs/v2.2/proposal.md`“背景与现状”。

**约束（必须满足）**：
- 技术栈不变：React + Ant Design + FastAPI + 文件存储
- 角色体系不变：admin / manager / expert / viewer
- 不引入 DB schema 迁移（见 REQ-C005）

**关键假设（含可验证方式）**：
- 假设 A1：历史报告版本能力已存在并可通过接口获取/下载（验证：`GET /api/v1/tasks/{task_id}/reports`、`GET /api/v1/tasks/{task_id}/reports/{report_id}`）。
- 假设 A2：ESB 导入能力已存在并可用（验证：`POST /api/v1/esb/imports`；且具备列识别/过滤/scope/统计能力）。
- 假设 A3：知识导入接口支持扩展 metadata 字段（如 `doc_type`）且不引入 DB 迁移（验证：向量库 metadata 可写入；或服务端可忽略未知字段并落入审计日志）。

### 1.3 术语与口径
- **效能看板**：侧边栏菜单“效能看板”，v2.2 下包含两个子页面：排行榜（`/dashboard/rankings`）与多维报表（`/dashboard/reports`）。
- **FUNC-022 / AI效果报告**：系统功能说明书中的“AI效果报告”能力与对应前端入口（`/dashboard?page=ai`、`/reports/ai-effect`）。v2.2 明确下线（见 REQ-012、REQ-C007）。
- **报告版本（ReportVersion）**：任务每轮生成的 PDF 版本；用户需可下载历史版本（见 REQ-006）。
- **实质性修改**：会影响 AI 评估结果的一类编辑操作（新增/删除功能点；修改核心字段；新增/重命名/删除系统 Tab），必须走“保存并确认”（见 REQ-007）。
- **ESB 特有能力**：列名自动识别/映射预览、状态过滤（正常/废弃）、scope(provider/consumer/both) 切换、统计（有效/废弃/去重接口数）（见 REQ-009）。
- **统计周期**：排行榜/看板统计窗口 v2.2 固定为近 90 天，不提供可配置入口（见 REQ-C008）。

## 2. 业务场景说明

### 2.1 角色与对象
- 角色
  - **admin**：查看全局数据与配置（看板/任务/系统清单等）
  - **manager**：创建/编辑任务、导入系统画像材料、触发重评估
  - **expert**：执行专家评估、查看任务与报告
  - **viewer**：只读查看看板与任务
- 对象
  - **Task**：评估任务（含评估轮次、报告版本、修改记录/备注等）
  - **Feature**：功能点（名称/描述/所属系统/预估人天等）
  - **ReportVersion**：报告版本（PDF）
  - **SystemProfile**：系统画像材料与字段
  - **ESBIndex**：ESB 集成索引（导入/检索/统计）
  - **Remark**：备注/变更摘要（AI 追加生成）

### 2.3 场景明细

#### SCN-001：查看排行榜
**场景分类**：看板  
**主要角色**：admin/manager/expert/viewer  
**相关对象**：看板统计数据  
**关联需求ID**：REQ-002、REQ-003、REQ-013  
**前置条件**：用户已登录且具备看板访问权限  
**流程步骤**：
1. 用户从侧边栏进入“效能看板→排行榜”
2. 用户在 3 个排行 Tab 中切换查看
3. 用户查看每个 Tab 右下角的计算逻辑说明
**输出产物**：排行榜列表与说明文案  
**异常与边界处理**：无数据时展示空状态；接口失败时展示错误提示与重试入口

#### SCN-002：查看多维报表
**场景分类**：看板  
**主要角色**：admin/manager/expert/viewer  
**相关对象**：看板统计数据  
**关联需求ID**：REQ-002、REQ-004、REQ-012  
**前置条件**：用户已登录且具备看板访问权限  
**流程步骤**：
1. 用户从侧边栏进入“效能看板→多维报表”
2. 用户查看 3 张卡片（总览统计/系统影响分析/流程健康度）
3. 用户确认管理驱动指标在对应卡片中可见
**输出产物**：多维报表卡片与指标展示  
**异常与边界处理**：无数据时展示空状态；接口失败时展示错误提示与重试入口

#### SCN-003：访问旧看板 URL 自动跳转
**场景分类**：看板  
**主要角色**：admin/manager/expert/viewer  
**相关对象**：无  
**关联需求ID**：REQ-013、REQ-012  
**前置条件**：用户已登录  
**流程步骤**：
1. 用户访问旧 URL（如 `/dashboard?page=rankings` 或 `/dashboard?page=ai`）
2. 系统将 URL 重写为新路由（`/dashboard/rankings` 或 `/dashboard/reports`）
3. 若为 `page=ai`，额外展示提示“AI效果报告已下线”
**输出产物**：新路由页面 +（可选）提示文案  
**异常与边界处理**：无

#### SCN-004：子菜单访问任务列表
**场景分类**：任务  
**主要角色**：admin/manager/expert/viewer  
**相关对象**：Task  
**关联需求ID**：REQ-005、REQ-013  
**前置条件**：用户已登录且具备任务列表访问权限  
**流程步骤**：
1. 用户从侧边栏进入“任务管理→进行中”查看进行中任务列表
2. 用户点击“任务管理→已完成”查看已完成任务列表
3. 若用户访问旧 URL（`/tasks?tab=completed`），系统自动跳转到 `/tasks/completed`
**输出产物**：任务列表页  
**异常与边界处理**：无数据展示空状态；接口失败展示错误提示与重试入口

#### SCN-005：查看任务详情并下载报告
**场景分类**：任务  
**主要角色**：admin/manager/expert/viewer（按任务权限）  
**相关对象**：Task、ReportVersion  
**关联需求ID**：REQ-006  
**前置条件**：用户对该任务具备访问权限  
**流程步骤**：
1. 用户进入任务详情页
2. 在摘要卡片查看关键字段（状态/创建时间/提交人/系统名/功能点数/专家进度/评估轮次）
3. 点击“下载报告”按钮下载最新报告；或通过下拉选择历史版本下载
**输出产物**：PDF 下载  
**异常与边界处理**：无报告版本时按钮置灰并提示“暂无可下载报告”；无权限返回 403 提示

#### SCN-006：PM 实质性修改确认并重评估
**场景分类**：编辑  
**主要角色**：manager  
**相关对象**：Task、Feature、Remark  
**关联需求ID**：REQ-007、REQ-011、REQ-C001、REQ-C002  
**前置条件**：manager 对任务具备编辑权限；任务处于允许编辑的状态  
**流程步骤**：
1. manager 进入功能点编辑页，对功能点执行实质性修改（如修改名称/描述/所属系统）
2. 点击“保存并确认”按钮
3. 系统弹出确认提示“本次修改将触发 AI 重新评估，是否确认？”
4. manager 点击“确认”后，系统保存变更并异步触发 AI 重评估
5. 重评估完成后，任务备注新增一条“AI重评估”摘要（只读展示）
**输出产物**：功能点变更落库；重评估作业；备注摘要  
**异常与边界处理**：用户取消确认不保存；重评估接口返回冲突/失败时提示并可重试

#### SCN-007：知识导入（代码扫描/文档导入）
**场景分类**：画像  
**主要角色**：manager  
**相关对象**：SystemProfile、ESBIndex  
**关联需求ID**：REQ-008、REQ-009、REQ-C006  
**前置条件**：manager 已登录；存在可操作系统（主责/B 角）；具备材料文件  
**流程步骤**：
1. manager 进入“系统画像→知识导入”，选择目标系统
2. 在“代码扫描”Tab 提交扫描并入库（如适用）
3. 切换到“文档导入”Tab，选择文档类型并上传文件
4. 当类型为“ESB接口文档”时，系统提供 ESB 专属导入表单与导入后检索/统计面板
**输出产物**：导入结果（success/failed/errors）；（ESB）检索/统计能力可用  
**异常与边界处理**：越权返回 403；文件格式不支持提示；embedding 服务不可用提示稍后重试

#### SCN-008：专家查看 COSMIC 规则
**场景分类**：评估  
**主要角色**：expert  
**相关对象**：无  
**关联需求ID**：REQ-010  
**前置条件**：expert 已登录且可访问评估任务  
**流程步骤**：
1. expert 进入专家评估页
2. 点击顶部“?”图标打开 COSMIC 规则详情
3. 关闭弹窗并继续评估
**输出产物**：规则弹窗展示  
**异常与边界处理**：无

#### SCN-009：备注自动生成与只读展示
**场景分类**：审计  
**主要角色**：admin/manager/expert/viewer（只读查看）  
**相关对象**：Remark  
**关联需求ID**：REQ-011  
**前置条件**：任务存在并有编辑/评估/重评估行为  
**流程步骤**：
1. 系统在关键动作后生成备注摘要（PM 编辑/专家评估完成/AI 重评估完成）
2. 任务详情/相关页面只读展示备注摘要，按时间倒序排列
**输出产物**：备注摘要列表  
**异常与边界处理**：无备注时展示“-”或空状态；不允许人工编辑入口

#### SCN-010：FUNC-022 下线与旧入口处理
**场景分类**：下线  
**主要角色**：admin/manager/expert  
**相关对象**：无  
**关联需求ID**：REQ-012、REQ-C007、REQ-013  
**前置条件**：用户已登录  
**流程步骤**：
1. 用户从旧入口访问 AI 效果报告（`/reports/ai-effect` 或 `/dashboard?page=ai`）
2. 系统自动跳转到 `/dashboard/reports`
3. 系统展示一次性提示“AI效果报告已下线”
**输出产物**：跳转与提示  
**异常与边界处理**：无

### 3.2 功能性需求明细

#### REQ-001：页面标题精简
**目标/价值**：减少页面冗余信息，提升阅读效率。  
**入口/触发**：访问对应页面  
**页面与交互**：
- 删除 PageHeader 标题的页面：
  - 任务列表（进行中/已完成）
  - 系统清单配置
  - COSMIC 规则配置
  - 用户管理
  - 知识库管理
  - 系统画像-知识导入
  - 系统画像-信息展示
  - 消息通知（`/notifications`）
**验收标准（GWT，必须可判定）**：

#### REQ-101：统一加载/空/错误态
**需求分类**：可用性  
**适用范围**：`/dashboard/rankings`、`/dashboard/reports`、`/tasks/*`、任务详情页、系统画像导入页等新增/重构页面

- [ ] GWT-REQ-101-03: Given 接口返回非 2xx 或网络错误，When 页面渲染，Then 页面展示错误提示且提供“重试”入口

## 6. 数据与接口

### 6.1 数据字典（增量/关键字段）
| 字段/对象 | 类型 | 必填 | 来源/去向 | 脱敏 | 留存期 | 备注 |
|---|---|---|---|---|---|---|
| `doc_type`（文档类型标记） | string(enum) | 否 | 前端 → `POST /api/v1/knowledge/imports` → metadata | 否 | 同知识库留存 | 仅对 4 类文档生效（非 ESB） |
| `remark`（备注展示） | text | 否 | 后端生成 → 前端只读展示 | 否 | 同任务留存 | v2.2 起 AI 追加摘要行；保留历史人工备注 |
| `report_versions` | array | 否 | 任务存储 → `GET /api/v1/tasks/{task_id}/reports` | 否 | 同任务留存 | 用于历史报告下拉下载 |

### 6.2 错误码与提示语（增量）
| 错误码 | 场景 | 提示语 | HTTP状态码 | 处理建议 |
|---|---|---|---|---|
| `AUTH_001` | 越权访问（导入/下载/任务访问） | 权限不足 | 403 | 提示联系管理员或切换账号 |
| `ESB_001`/`ESB_002` | ESB 文件解析/必填字段缺失 | 文件格式不支持/缺少必填字段 | 400 | 提示用户检查文件格式/列名 |
| `KNOW_001`/`KNOW_002` | 知识导入失败 | 知识库文件类型不支持/解析失败 | 400 | 提示用户检查文件类型/内容 |
| `EMB_001` | embedding 服务不可用 | embedding服务不可用，请稍后重试 | 503 | 提示稍后重试 |

### 6.3 指标与计算口径（排行榜）
- **评估效率排行**：评估效率 = 已完成评估数 / 累计评估耗时（天），统计窗口：近 90 天
- **任务提交排行**：近 90 天内各项目经理提交任务总数（降序）
- **系统活跃排行**：近 90 天内各系统关联评估任务数（降序）
