## 1. 快速开始

### 1.1 登录/访问方式

**系统访问地址**：`http://<服务器IP>/`

**默认账号**：

| 角色 | 用户名 | 密码 | 说明 |
|---|---|---|---|
| 管理员 | admin | admin123 | 系统管理员，拥有所有权限 |
| 项目经理 | manager | manager123 | 可发起评估任务 |
| 专家 | expert1 | expert123 | 评估专家（还有expert2、expert3） |

> 生产环境请修改默认密码！

### 1.2 基本概念

**角色切换（多角色账号）**：
- 右上角头像区域支持角色切换（如“项目经理 ↔ 评估专家”）
- 登录后默认按权限优先级激活角色：管理员 > 项目经理 > 专家
- 切换后菜单与数据范围立即生效，无需重新登录

**系统角色**：
- **管理员**：负责系统配置、用户管理、查看所有任务
- **管理层/查看者（Viewer）**：只读查看效能看板与任务明细下钻（不做配置与编辑）
- **项目经理**：发起评估任务、编辑功能点、维护系统画像与知识导入、查看报告
- **专家**：对功能点进行人天评估、参与多轮评估

**核心流程**：
```
上传需求文档 → AI分析 → 复核修正 → 专家评估 → 生成报告
```

**系统识别分层**：
- **selected_systems**：确定涉及的系统（证据充分）
- **maybe_systems**：可能涉及的系统（证据不足）
- **questions**：需要澄清的问题集

**证据等级**：
- **E3（高）**：具备代码扫描 + 证据/画像
- **E2（中）**：具备证据/画像 + ESB或代码
- **E1（低）**：仅具备ESB或证据
- **E0（无）**：无可用证据

### 1.3 v2.1 变化速览

## 2. 功能使用指南

### 2.1 项目经理使用指南

#### 2.1.1 发起评估任务

**步骤**：
1. 登录系统后，点击左侧菜单「任务管理」
2. 点击右上角「新建任务/上传需求」
3. 选择需求文档（支持 `.docx/.doc/.xls`，最大10MB）
4. 点击「开始评估」
5. 等待AI处理完成（约1-5分钟）
6. 处理完成后自动跳转到任务列表

**注意事项**：
- 需求文档必须包含"需求内容说明"章节
- 文档大小不能超过10MB
- AI处理期间请勿关闭页面

#### 2.1.2 编辑评估结果

**步骤**：
1. 在任务列表找到要编辑的任务
2. 点击「编辑」按钮
3. 查看AI识别结果（selected_systems / maybe_systems / questions）
4. 切换系统Tab查看功能点
5. 可以进行以下操作：
   - 新增/删除/编辑功能点
   - 新增/删除/重命名系统Tab
   - 重新拆分单个系统
6. 点击「保存」保存修改
7. 确认无误后点击「提交给管理员」

**可编辑字段**：
- 功能模块、功能点、业务描述
- 输入、输出、依赖项

**不可编辑字段**（AI生成，仅供参考）：
- 预估人天数、置信度、假设、风险标识、证据等级

#### 2.1.3 查看报告

**步骤**：
1. 在任务列表找到已完成的任务
2. 点击「下载报告」
3. 可选择下载不同轮次的报告版本

#### 2.1.4 维护知识库

**证据库（A层）**：
1. 点击左侧菜单「系统画像」→「知识导入」
2. 选择目标系统
3. 在「知识导入」区域上传文件（DOC/DOCX/XLS/XLSX/PDF/TXT/CSV，按页面提示为准）
4. 按需选择导入类型（normal/L0）与系统绑定信息
5. 点击「导入」（导入成功后会刷新材料完整度）

**系统画像（B层）**：
1. 点击左侧菜单「系统画像」→「信息看板」
2. 选择目标系统（系统TAB）
3. 编辑系统画像字段（4项）：
   - `system_scope`（系统范围）
   - `module_structure`（模块结构）
   - `integration_points`（主要集成点）
   - `key_constraints`（关键约束）
4. 点击「保存草稿」或「发布」
> 说明：发布校验至少要求 `system_scope` 与 `module_structure` 非空。

**ESB索引（I层）**：
1. 点击左侧菜单「系统画像」→「知识导入」
2. 选择目标系统（系统TAB）
3. 在「ESB导入」区域上传 CSV/XLSX 文件（可选填写 `mapping_json` 做字段映射）
4. 查看“本次导入统计”（不展示导入历史列表）

**代码扫描（C层）**：
1. 点击左侧菜单「系统画像」→「知识导入」
2. 选择目标系统（系统TAB）
3. 在「代码扫描」区域填写仓库本地路径或上传仓库压缩包
4. 点击「提交扫描」，等待扫描完成
5. 点击「提交入库」将能力目录写入知识库（同时更新材料完整度）

### 2.2 管理员使用指南

#### 2.2.1 查看所有任务

**步骤**：
1. 登录系统后，点击「任务管理」
2. 可查看所有任务（包括其他用户发起的）
3. 任务列表按状态分组：
   - 进行中：草稿 / 待分配 / 评估中
   - 已完成：已完成 / 已归档

#### 2.2.2 分配专家

**步骤**：
1. 在任务列表找到待分配的任务
2. 点击「分配专家」
3. 选择3位在岗专家
4. 点击「确认分配」

**注意事项**：
- 必须选择3位专家
- 专家必须在岗且状态为启用

#### 2.2.3 系统清单管理

**步骤**：
1. 点击「配置管理」→「系统清单」
2. 切换「主系统清单」Tab：
   - 增删改主系统
   - 批量导入XLSX
3. 切换「子系统映射」Tab：
   - 增删改子系统映射
   - 批量导入XLSX

#### 2.2.4 用户管理

**步骤**：
1. 点击「配置管理」→「用户管理」
2. 点击「新增用户」添加单个用户
3. 或点击「批量导入」：
   - 下载Excel模板
   - 填写用户信息
   - 上传文件
   - 预览并确认

### 2.3 专家使用指南

#### 2.3.1 接收评估邀请

**步骤**：
1. 管理员分配专家后，会收到通知
2. 点击「消息通知」查看邀请
3. 点击通知中的链接进入评估页面

#### 2.3.2 填写评估

**步骤**：
1. 进入评估页面后，查看功能点列表
2. 灰色数值为AI预估值（默认值）
3. 点击「预估人天数」单元格进行编辑
4. 输入您的评估值
5. 失焦自动保存
6. 完成后点击「提交评估」

**注意事项**：
- 未修改的值将以AI预估作为您的评估结果
- 第二/三轮评估仅展示高偏离功能点（红色高亮）
- 未生成报告前可以撤回

#### 2.3.3 查看我的评估

**步骤**：
1. 点击「任务管理」
2. 按「进行中/已完成」Tab 查看我参与的评估任务
3. 点击「查看」查看评估详情

## 3. 权限与角色

### 3.1 角色说明

| 角色 | 主要职责 |
|---|---|
| **管理员** | 系统配置、用户管理、查看所有任务、分配专家 |
| **查看者（Viewer）** | 只读查看效能看板与任务下钻列表（不做配置/编辑） |
| **项目经理** | 发起评估任务、编辑功能点、维护系统画像与知识导入、查看报告 |
| **专家** | 对功能点进行人天评估、查看评估结果 |

### 3.2 权限矩阵

| 功能 | 管理员 | 查看者（Viewer） | 项目经理 | 专家 |
|---|---|---|---|---|
| 查看任务明细（只读） | ✅ | ✅（全局） | ✅（创建者/主责范围） | ✅（参与任务） |
| 发起评估任务 | ❌ | ❌ | ✅ | ❌ |
| 分配专家 | ✅ | ❌ | ❌ | ❌ |
| 编辑功能点 | ❌ | ❌ | ✅ | ❌ |
| 评估人天 | ❌ | ❌ | ❌ | ✅ |
| 查看报告 | ✅ | ❌ | ✅ | ✅（参与任务） |
| 系统清单管理 | ✅ | ❌ | ❌ | ❌ |
| 用户管理 | ✅ | ❌ | ❌ | ❌ |
| 系统画像与知识导入 | 只读 | ❌ | ✅（主责/B角可写） | 只读 |
| 效能看板 | ✅ | ✅ | ✅（范围按权限） | ✅（范围按权限） |

### 3.3 常见权限问题

**Q: 为什么看不到「分配专家」按钮？**
A: 只有管理员角色才能分配专家。请联系管理员。

**Q: 为什么不能编辑功能点？**
A: 只有项目经理角色可以编辑功能点。任务提交给管理员后功能点锁定。

**Q: 专家为什么看不到某些任务？**
A: 专家只能看到管理员分配给自己的评估任务。

## 4. 常见问题（FAQ）

### 4.1 登录相关

**Q: 忘记密码怎么办？**
A: 请联系管理员重置密码。

**Q: 首次登录后建议修改密码？**
A: 是的，建议首次登录后立即修改密码。点击右上角用户名→「个人中心」→「密码安全」。

### 4.2 任务相关

**Q: AI处理失败怎么办？**
A: 任务会保留在草稿状态，可以检查文档格式后重新上传。

**Q: 评估中可以修改功能点吗？**
A: 不可以。任务提交给管理员后进入评估流程，功能点锁定。

**Q: 专家评估可以撤回吗？**
A: 在报告生成前可以撤回并重新评估。

**Q: 为什么有些系统进入maybe？**
A: 说明证据不足或存在冲突。建议补充相关证据材料或画像。

### 4.3 报告相关

**Q: 报告可以修改吗？**
A: 报告是评估结果的快照，不可直接修改。如需调整，需要重新发起评估。

**Q: 历史报告会丢失吗？**
A: 不会。系统会保存所有历史版本的报告。

### 4.4 AI相关

**Q: AI识别不准确怎么办？**
A: 项目经理可以在编辑页修正AI结果，修正数据会用于后续AI优化。

**Q: 证据等级是什么？**
A: 反映系统/功能点在本次任务中可用证据的综合等级（E0~E3），用于风险提示与阻断判断。

**Q: 什么是严肃模式？**
A: 证据不足时禁止输出确定结论，仅输出 `maybe + questions`，用于采购严肃场景的风险控制。

### 4.5 技术问题

**Q: 上传文档失败？**
A: 确保为 `.docx/.doc/.xls` 且不超过10MB。

**Q: 证据导入失败？**
A: 确保PDF非扫描版，或改用 `.doc/.docx/.xls/.xlsx` 等可解析格式。

**Q: 无法预览证据？**
A: 检查是否参与该任务，专家只能预览其参与任务关联的证据。
