## 1. 快速开始

### 1.1 登录/访问方式

**系统访问地址**：`http://<服务器IP>/`

**默认账号**：

| 角色 | 用户名 | 密码 | 说明 |
|---|---|---|---|
| 管理员 | admin | admin123 | 系统管理员，拥有所有权限 |
| 项目经理 | manager | manager123 | 可发起评估任务 |
| 专家 | expert1 | expert123 | 评估专家（还有expert2、expert3） |

> 生产环境请修改默认密码！

### 1.2 基本概念

**角色切换（多角色账号）**：
- 右上角头像区域支持角色切换（如“项目经理 ↔ 评估专家”）
- 登录后默认按权限优先级激活角色：管理员 > 项目经理 > 专家
- 切换后菜单与数据范围立即生效，无需重新登录

**系统角色**：
- **管理员**：负责系统配置、用户管理、查看所有任务
- **管理层/查看者（Viewer）**：只读查看效能看板与任务明细下钻（不做配置与编辑）
- **项目经理**：发起评估任务、编辑功能点、维护系统画像与知识导入、查看报告
- **专家**：对功能点进行人天评估、参与多轮评估

**核心流程**：
```
上传需求文档 → AI分析 → 复核修正 → 专家评估 → 生成报告
```

**系统识别分层**：
- **selected_systems**：确定涉及的系统（证据充分）
- **maybe_systems**：可能涉及的系统（证据不足）
- **questions**：需要澄清的问题集

**证据等级**：
- **E3（高）**：具备代码扫描 + 证据/画像
- **E2（中）**：具备证据/画像 + ESB或代码
- **E1（低）**：仅具备ESB或证据
- **E0（无）**：无可用证据

### 1.3 v2.2 变化速览
- 效能看板改为左侧子菜单双入口：`排行榜`（`/dashboard/rankings`）与`多维报表`（`/dashboard/reports`）。
- 任务管理改为左侧子菜单双入口：`进行中`（`/tasks/ongoing`）与`已完成`（`/tasks/completed`）。
- PM 编辑增加“保存并确认”门禁：实质性修改需确认后触发重评估；`预估人天`对 PM 只读。
- 系统画像子菜单文案更新为“知识导入 / 信息展示”。
- `AI效果报告`入口下线；访问旧链接会跳转到`/dashboard/reports`并提示“AI效果报告已下线”。

### 1.4 v2.3 变化速览
- 代码扫描新增 `repo_source` 来源模式：`local`、`archive`、`gitlab_archive`、`gitlab_compare`、`gitlab_raw`。
- GitLab Compare/Raw 等来源参数门禁统一，缺参或冲突返回 `SCAN_007`，提示补全模式必填参数。
- 扫描结果在原有能力条目外新增 `analysis` 与 `metrics` 摘要字段，供评估证据与回归统计复用。

### 1.5 v2.4 变化速览
- 「知识导入」页改为文档类型卡片化独立操作区，不同类型互不清空状态，并新增导入历史列表与“查看系统画像”快捷跳转。
- 「信息展示」页升级为三区布局（域导航 + 内容区 + 时间线），系统画像从 4 字段升级为 5 域 12 子字段结构。
- 画像支持子字段级 AI 建议对比（inline diff），可按字段执行“采纳/忽略/恢复上一版建议”。
- 评估页与报告页支持三点估计展示（`expected` 主展示，展开查看 O/M/P + 估算理由），Excel 导出新增三点估计列。
- 评估流程新增 AI 原始输出快照与 PM 修正 diff 查询能力，用于后续经验沉淀与回溯。

### 1.6 v2.5 变化速览
- 「信息展示」页按 D1-D5 五域结构化重构：D2 支持 3 层树形与折叠/展开，超过 100 节点给出阈值提示；D3/D5 采用表格化展示。
- AI 建议改为混合 diff 展示：文本、列表、表格、树形字段均可直接对比变化点。
- 「知识导入」页新增模板下载（历史评估报告/ESB 文档），上传前可直接下载标准模板。
- 导入状态默认走 WebSocket 实时推送；连接失败自动降级为 5 秒轮询（最多 10 次），导入完成自动刷新画像。
- WebSocket 心跳固定 30 秒（`ping/pong`）；异常断链时页面不阻塞，可继续操作。

### 1.7 v2.7 变化速览
- 「知识导入」页的文档卡片收敛为 `需求文档 / 设计文档 / 技术方案` 三类，代码扫描入口保留为独立卡片；不再展示历史评估报告或治理文档入口。
- 导入状态主查询接口改为 `profile/execution-status`；页面仍兼容旧 `extraction-status` 数据来源。
- 「信息展示」页保持原有“系统TAB + 5个域TAB”交互，只展示 D1-D5 canonical 内容与 AI 建议；扩展信息、Memory 资产、来源/操作人仅在后台留存，不在前端展示。
- 管理员新增「服务治理」页（`/admin/service-governance`），用于全局导入治理文件并查看命中/跳过结果。
- 「系统清单」页改为单页单模板，不再区分“主系统清单/子系统映射”Tab；confirm 后只初始化空画像，非空画像会在结果中标记跳过原因。

## 2. 功能使用指南

### 2.1 项目经理使用指南

#### 2.1.1 发起评估任务

**步骤**：
1. 登录系统后，点击左侧菜单「任务管理」
2. 点击右上角「新建任务/上传需求」
3. 选择需求文档（支持 `.docx/.doc/.xls`，最大10MB）
4. 点击「开始评估」
5. 等待AI处理完成（约1-5分钟）
6. 处理完成后自动跳转到任务列表

**注意事项**：
- 需求文档必须包含"需求内容说明"章节
- 文档大小不能超过10MB
- AI处理期间请勿关闭页面

#### 2.1.2 编辑评估结果

**步骤**：
1. 在任务列表找到要编辑的任务
2. 点击「编辑」按钮
3. 查看AI识别结果（selected_systems / maybe_systems / questions）
4. 切换系统Tab查看功能点
5. 可以进行以下操作：
   - 新增/删除/编辑功能点
   - 新增/删除/重命名系统Tab
   - 重新拆分单个系统
6. 点击「保存」保存修改
7. 确认无误后点击「提交给管理员」

**可编辑字段**：
- 功能模块、功能点、业务描述
- 输入、输出、依赖项

**不可编辑字段**（AI生成，仅供参考）：
- 预估人天数、置信度、假设、风险标识、证据等级

#### 2.1.3 查看报告

**步骤**：
1. 在任务列表找到已完成的任务
2. 点击「下载报告」
3. 可选择下载不同轮次的报告版本

#### 2.1.4 维护知识库

**证据库（A层）**：
1. 点击左侧菜单「系统画像」→「知识导入」
2. 选择目标系统
3. 在「知识导入」区域上传文件（DOC/DOCX/XLS/XLSX/PDF/TXT/CSV，按页面提示为准）
4. 按需选择导入类型（normal/L0）与系统绑定信息
5. 点击「导入」（导入成功后会刷新材料完整度）

**PM 文档导入（v2.7）**：
1. 在「知识导入」页选择目标系统
2. 在文档卡片中仅选择 `需求文档 / 设计文档 / 技术方案`
3. 上传文件后查看“执行状态”，状态以 `排队中 / 执行中 / 已完成 / 部分成功 / 失败` 展示
4. 导入成功后可在导入历史中查看执行时间和结果，并跳转到「信息展示」页核对画像变化

**系统画像（B层）**：
1. 点击左侧菜单「系统画像」→「信息展示」
2. 选择目标系统（系统TAB）
3. 在左侧域导航中选择要编辑的域（系统定位与边界、业务能力、集成与接口、技术架构、约束与风险）
4. 在中间内容区编辑对应子字段（共 12 个子字段，按域分组展示）
5. 若页面展示 AI 建议差异，可按子字段执行「采纳」「忽略」「恢复上一版建议」
6. 在右侧时间线查看变更事件（导入、采纳、回滚、手动编辑等）
7. 点击「保存草稿」或「发布」
> 说明：发布校验以页面实时提示为准；首次导入前“恢复上一版建议”会置灰。

**ESB索引（I层）**：
1. 点击左侧菜单「系统画像」→「知识导入」
2. 选择目标系统（系统TAB）
3. 在「ESB导入」区域上传 CSV/XLSX 文件（可选填写 `mapping_json` 做字段映射）
4. 查看“本次导入统计”（不展示导入历史列表）

**代码扫描（C层）**：
1. 点击左侧菜单「系统画像」→「知识导入」
2. 选择目标系统（系统TAB）
3. 在「代码扫描」区域选择 `repo_source`（`local/archive/gitlab_archive/gitlab_compare/gitlab_raw`）
4. 按来源模式填写参数（本地路径、归档包或 GitLab Compare/Raw 参数）
5. 点击「提交扫描」，等待扫描完成
6. 若提示 `SCAN_007`，按当前 `repo_source` 补齐必填参数后重试
7. 点击「提交入库」将能力目录写入知识库（同时更新材料完整度）

### 2.2 管理员使用指南

#### 2.2.1 查看所有任务

**步骤**：
1. 登录系统后，点击「任务管理」
2. 可查看所有任务（包括其他用户发起的）
3. 任务列表按状态分组：
   - 进行中：草稿 / 待分配 / 评估中
   - 已完成：已完成 / 已归档

#### 2.2.2 分配专家

**步骤**：
1. 在任务列表找到待分配的任务
2. 点击「分配专家」
3. 选择3位在岗专家
4. 点击「确认分配」

**注意事项**：
- 必须选择3位专家
- 专家必须在岗且状态为启用

#### 2.2.3 系统清单管理

**步骤**：
1. 点击「配置管理」→「系统清单」
2. 在单页中维护系统清单或下载最新模板
3. 上传 XLSX 后先查看 preview 结果
4. 确认无误后执行 confirm，系统会返回：
   - 已更新系统：本次完成空画像初始化的系统名称
   - 跳过项：因画像已有内容等原因未初始化的系统
   - 预检错误/异常：需回到模板中修正的行级问题

**注意事项**：
- 页面已不再区分“主系统清单 / 子系统映射”Tab
- confirm 不会粗暴覆盖已有画像；只有 D1-D5 canonical 全空的系统才会被初始化

#### 2.2.4 服务治理

**步骤**：
1. 点击左侧菜单「服务治理」
2. 上传治理文件（Excel/CSV，按模板要求）
3. 查看导入结果中的匹配成功数、未匹配数、已更新系统名称
4. 对未命中的系统名称回到系统清单核对标准名称后再重试

**注意事项**：
- 只有管理员可以执行全局治理导入
- 治理导入以系统标准名称精确匹配为准，不做手工模糊映射
- 关联系统等弱证据信息不会直接改写 D3 canonical，而是作为受控建议或后台留存

#### 2.2.5 用户管理

**步骤**：
1. 点击「配置管理」→「用户管理」
2. 点击「新增用户」添加单个用户
3. 或点击「批量导入」：
   - 下载Excel模板
   - 填写用户信息
   - 上传文件
   - 预览并确认

### 2.3 专家使用指南

#### 2.3.1 接收评估邀请

**步骤**：
1. 管理员分配专家后，会收到通知
2. 点击「消息通知」查看邀请
3. 点击通知中的链接进入评估页面

#### 2.3.2 填写评估

**步骤**：
1. 进入评估页面后，查看功能点列表
2. 灰色数值为AI预估值（默认值）
3. 点击「预估人天数」单元格进行编辑
4. 输入您的评估值
5. 失焦自动保存
6. 完成后点击「提交评估」

**注意事项**：
- 未修改的值将以AI预估作为您的评估结果
- 第二/三轮评估仅展示高偏离功能点（红色高亮）
- 未生成报告前可以撤回

#### 2.3.3 查看我的评估

**步骤**：
1. 点击「任务管理」
2. 通过左侧子菜单「进行中/已完成」查看我参与的评估任务
3. 点击「查看」查看评估详情

| 角色 | 主要职责 |

| **项目经理** | 发起评估任务、编辑功能点、维护系统画像与知识导入、查看报告 |

### 3.3 常见权限问题

**Q: 为什么看不到「分配专家」按钮？**
A: 只有管理员角色才能分配专家。请联系管理员。

**Q: 为什么不能编辑功能点？**
A: 只有项目经理角色可以编辑功能点。任务提交给管理员后功能点锁定。

**Q: 专家为什么看不到某些任务？**
A: 专家只能看到管理员分配给自己的评估任务。

## 4. 常见问题（FAQ）

### 4.1 登录相关

**Q: 忘记密码怎么办？**
A: 请联系管理员重置密码。

**Q: 首次登录后建议修改密码？**
A: 是的，建议首次登录后立即修改密码。点击右上角用户名→「个人中心」→「密码安全」。

### 4.2 任务相关

**Q: AI处理失败怎么办？**
A: 任务会保留在草稿状态，可以检查文档格式后重新上传。

**Q: 评估中可以修改功能点吗？**
A: 不可以。任务提交给管理员后进入评估流程，功能点锁定。

**Q: 专家评估可以撤回吗？**
A: 在报告生成前可以撤回并重新评估。

**Q: 为什么有些系统进入maybe？**
A: 说明证据不足或存在冲突。建议补充相关证据材料或画像。

### 4.3 报告相关

**Q: 报告可以修改吗？**
A: 报告是评估结果的快照，不可直接修改。如需调整，需要重新发起评估。

**Q: 历史报告会丢失吗？**
A: 不会。系统会保存所有历史版本的报告。

### 4.4 AI相关

**Q: AI识别不准确怎么办？**
A: 项目经理可以在编辑页修正AI结果，修正数据会用于后续AI优化。

**Q: 证据等级是什么？**
A: 反映系统/功能点在本次任务中可用证据的综合等级（E0~E3），用于风险提示与阻断判断。

**Q: 什么是严肃模式？**
A: 证据不足时禁止输出确定结论，仅输出 `maybe + questions`，用于采购严肃场景的风险控制。

### 4.5 技术问题

**Q: 上传文档失败？**
A: 确保为 `.docx/.doc/.xls` 且不超过10MB。

**Q: 证据导入失败？**
A: 确保PDF非扫描版，或改用 `.doc/.docx/.xls/.xlsx` 等可解析格式。

**Q: 无法预览证据？**
A: 检查是否参与该任务，专家只能预览其参与任务关联的证据。
