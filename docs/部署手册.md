# 部署手册

## 一、环境要求

### 1.1 硬件要求
- CPU: 4核以上（推荐8核）
- 内存: 8GB以上（推荐16GB，启用Milvus知识库需要更多）
- 磁盘: 20GB以上可用空间

### 1.2 软件要求
- **操作系统**: Linux (推荐Ubuntu 20.04+, CentOS 7+)
- **Python**: 3.10+
- **Node.js**: 18+ (前端构建)
- **Docker**: 20.10+ (推荐)
- **Docker Compose**: 2.0+ (推荐)
- **Nginx**: 1.18+ (生产环境)

### 1.3 外部依赖
- **阿里云DashScope API**: 需要申请API Key
- **Milvus向量数据库**: 可选，用于知识库功能

## 二、快速部署（Docker Compose）

### 2.1 准备工作

#### 2.1.1 安装Docker和Docker Compose

**Ubuntu/Debian**:
```bash
# 更新包索引
sudo apt-get update

# 安装依赖
sudo apt-get install -y apt-transport-https ca-certificates curl gnupg lsb-release

# 添加Docker官方GPG key
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 设置Docker仓库
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装Docker Engine
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 验证安装
docker --version
docker compose version
```

**CentOS/RHEL**:
```bash
# 安装Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动Docker
sudo systemctl start docker
sudo systemctl enable docker
```

#### 2.1.2 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑.env文件
vi .env
```

**必需配置**:
```bash
# 阿里云大模型配置（必需）
DASHSCOPE_API_KEY=sk-your-api-key-here
DASHSCOPE_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=qwen-turbo
```

**可选配置**:
```bash
# Milvus向量数据库（启用知识库需要）
MILVUS_HOST=localhost
MILVUS_PORT=19530
KNOWLEDGE_ENABLED=true

# CORS配置
ALLOWED_ORIGINS=http://your-frontend-url,http://another-url

# Delphi配置
DELPHI_ROUNDS=3
```

### 2.2 一键部署

#### 2.2.1 构建并启动服务

```bash

git clone <repository-url>
cd requirement-estimation-system

# 配置环境变量
cp .env.example .env
```

```bash
# 检查容器状态
docker-compose ps

# 检查后端健康状态
curl http://localhost:443/api/v1/health

# 预期输出
{
  "status": "healthy",
  "service": "业务需求工作量评估系统",
  "version": "1.0.0"
}
```

#### 2.2.3 服务访问

- **前端**: http://localhost (端口80)
- **后端API**: http://localhost:443 (端口443)
- **API文档**: http://localhost:443/docs

### 2.3 常用操作

```bash
# 停止服务
docker-compose down

# 重启服务
docker-compose restart

# 查看日志
docker-compose logs -f backend
```

## 五、生产环境部署

### 5.1 使用uv部署（推荐）

#### 5.1.1 安装uv

```bash
pip install uv
```

#### 5.1.2 创建虚拟环境

```bash
cd /path/to/requirement-estimation-system

# uv会自动创建和管理虚拟环境
uv sync
```

#### 5.1.3 配置systemd服务

创建服务文件 `/etc/systemd/system/requirement-estimation.service`:

```ini
[Unit]
Description=Requirement Estimation System
After=network.target milvus.service

[Service]
Type=notify
User=www-data
Group=www-data
WorkingDirectory=/app
Environment="PATH=/app/.venv/bin"
EnvironmentFile=-/app/.env
ExecStart=/usr/local/bin/uv run uvicorn backend.app:app --host 0.0.0.0 --port 443 --workers 4
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重载systemd配置
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start requirement-estimation

# 设置开机自启
sudo systemctl enable requirement-estimation

# 查看状态
sudo systemctl status requirement-estimation

# 查看日志
sudo journalctl -u requirement-estimation -f
```

### 5.2 使用Nginx反向代理

#### 5.2.1 安装Nginx

```bash
sudo apt-get install nginx
```

#### 5.2.2 配置Nginx

创建配置文件 `/etc/nginx/sites-available/requirement-estimation`:

```nginx
# 后端API服务器
server {
    listen 443 ssl;
    server_name your-domain.com;

    # SSL证书配置
    ssl_certificate /etc/nginx/ssl/cert.pem;
    ssl_certificate_key /etc/nginx/ssl/key.pem;

    # 文件上传大小限制
    client_max_body_size 10M;

    # API代理
    location /api/ {
        proxy_pass http://127.0.0.1:443;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket支持（如果需要）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 前端静态文件
    location / {
        root /var/www/requirement-estimation/frontend/build;
        try_files $uri $uri/ /index.html;
    }

    # 报告下载
    location /reports/ {
        alias /app/data/;
        add_header Content-Disposition "attachment";
        add_header Content-Type "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet";
    }
}

# HTTP重定向到HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

启用配置：

```bash
# 创建符号链接
sudo ln -s /etc/nginx/sites-available/requirement-estimation /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重载Nginx
sudo systemctl reload nginx
```

### 5.3 配置SSL证书

#### 使用Let's Encrypt（推荐）

```bash
# 安装certbot
sudo apt-get install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

#### 使用自签名证书（测试环境）

```bash
# 创建证书目录
sudo mkdir -p /etc/nginx/ssl

# 生成自签名证书
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/key.pem \
  -out /etc/nginx/ssl/cert.pem
```

## 八、日志管理

### 8.1 日志配置

日志文件位置：
- **应用日志**: `logs/app.log`
- **错误日志**: `logs/error.log`

### 8.2 查看日志

```bash
# 实时查看日志
tail -f logs/app.log

# 查看最近100行
tail -n 100 logs/app.log

# 搜索错误日志
grep "ERROR" logs/app.log

# 查看特定任务的日志
grep "task_id" logs/app.log | grep "f47ac10b"
```

### 8.3 日志轮转

使用logrotate管理日志：

```bash
# 创建logrotate配置
sudo vi /etc/logrotate.d/requirement-estimation
```

配置内容：

```
/path/to/requirement-estimation-system/logs/*.log {
    daily
    rotate 30
    compress
    delaycompress
    notifempty
    create 0640 www-data www-data
    sharedscripts
    postrotate
        systemctl reload requirement-estimation > /dev/null 2>&1 || true
    endscript
}
```

测试配置：

```bash
sudo logrotate -d /etc/logrotate.d/requirement-estimation
sudo logrotate -f /etc/logrotate.d/requirement-estimation
```

## 十、备份策略

### 10.1 需要备份的数据

- 上传的文档：`uploads/`
- 生成的报告：`data/`
- 修正历史：`data/task_modifications.json`
- 应用日志：`logs/`

### 10.2 自动备份脚本

创建备份脚本 `backup.sh`:

```bash
#!/bin/bash
BACKUP_DIR="/backup/requirement-estimation"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据
tar -czf $BACKUP_DIR/backup_$DATE.tar.gz \
  uploads/ \
  data/ \
  logs/ \
  .env

# 删除30天前的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +30 -delete

echo "Backup completed: backup_$DATE.tar.gz"
```

设置定时任务：

```bash
# 添加到crontab
crontab -e

# 每天凌晨2点执行备份
0 2 * * * /path/to/backup.sh
```

## 十一、故障排查

### 11.1 常见问题

#### 问题1：容器启动失败

**症状**：`docker-compose up` 失败

**排查步骤**：

```bash
# 查看详细日志
docker-compose logs backend
docker-compose logs frontend

# 检查端口占用
sudo netstat -tunlp | grep -E '443|80'

# 检查磁盘空间
df -h

# 重新构建
docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

#### 问题2：LLM调用失败

**症状**：日志显示 "LLM request failed"

**解决方案**：

1. 检查API Key配置
```bash
# 检查.env文件
cat .env | grep DASHSCOPE_API_KEY
```

2. 验证API Key
```bash
curl -X POST https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions \
  -H "Authorization: Bearer $DASHSCOPE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"qwen-turbo","messages":[{"role":"user","content":"test"}]}'
```

3. 检查网络连接
```bash
ping dashscope.aliyuncs.com
```

4. 查看详细错误日志
```bash
docker-compose logs backend | grep -i "llm\|api\|error"
```

#### 问题3：Milvus连接失败

**症状**：知识库功能不可用

**解决方案**：

```bash
# 检查Milvus容器状态
docker ps | grep milvus

# 测试Milvus连接
curl http://localhost:19530/healthz

# 检查Milvus日志
docker logs milvus-standalone

# 重启Milvus
docker restart milvus-standalone
```

#### 问题4：文件上传失败

**症状**：上传时报500错误

**解决方案**：

```bash
# 检查文件权限
ls -la uploads/
sudo chown -R www-data:www-data uploads/

# 检查磁盘空间
df -h

# 查看后端日志
docker-compose logs backend | tail -50
```

#### 问题5：前端无法访问后端

**症状**：前端报跨域错误

**解决方案**：

```bash
# 检查CORS配置
cat .env | grep ALLOWED_ORIGINS

# 修改配置后重启
docker-compose restart backend
```
