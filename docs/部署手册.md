# 部署手册

## 一、环境要求

### 1.1 硬件要求
- CPU: 4核以上
- 内存: 8GB以上
- 磁盘: 20GB以上可用空间

### 1.2 软件要求
- 操作系统: Linux/MacOS/Windows
- Python: 3.11+
- Node.js: 18+
- Docker: 20.10+ (可选)
- Docker Compose: 2.0+ (可选)

---

## 二、开发环境搭建

### 2.1 克隆项目

```bash
git clone <repository-url>
cd requirement-estimation-system
```

### 2.2 后端环境搭建

```bash
cd backend

# 创建虚拟环境（推荐）
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt

# 安装uv（推荐的Python运行工具）
pip install uv
```

### 2.3 前端环境搭建

```bash
cd frontend

# 安装依赖
npm install
```

### 2.4 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑.env文件，填入阿里云API Key
vi .env
```

### 2.5 启动服务

**启动后端**：
```bash
cd backend
uv app:app --host 0.0.0.0 --port 8000 --reload
```

**启动前端**：
```bash
cd frontend
npm start
```

访问：
- 前端: http://localhost:3000
- API文档: http://localhost:8000/docs

## 三、生产环境部署

### 3.1 使用Docker Compose部署（推荐）

#### 3.1.1 准备工作

1. 安装Docker和Docker Compose
2. 准备阿里云API Key
3. 配置环境变量文件

#### 3.1.2 部署步骤

```bash
# 1. 构建镜像
docker-compose build

# 2. 启动服务
docker-compose up -d

# 3. 查看日志
docker-compose logs -f

# 4. 停止服务
docker-compose down
```

#### 3.1.3 服务访问

- 前端: http://localhost:3000
- 后端API: http://localhost:8000
- API文档: http://localhost:8000/docs

### 3.2 使用uv部署（推荐用于生产）

#### 3.2.1 安装uv

```bash
pip install uv
```

#### 3.2.2 启动服务

```bash
cd backend

# 开发模式（单进程，自动重载）
uv app:app --host 0.0.0.0 --port 8000 --reload

# 生产模式（多进程）
uv app:app --host 0.0.0.0 --port 8000 --workers 4
```

#### 3.2.3 使用systemd管理（Linux）

创建服务文件 `/etc/systemd/system/requirement-estimation.service`:

```ini
[Unit]
Description=Requirement Estimation System
After=network.target

[Service]
Type=notify
User=www-data
WorkingDirectory=/app
Environment="PATH=/app/venv/bin"
ExecStart=/app/venv/bin/uv app:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务：
```bash
sudo systemctl daemon-reload
sudo systemctl start requirement-estimation
sudo systemctl enable requirement-estimation
```

## 四、Nginx配置（生产环境推荐）

### 4.1 反向代理配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /var/www/requirement-estimation/frontend/build;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # 文件上传大小限制
        client_max_body_size 10M;
    }

    # 文件下载
    location /reports/ {
        proxy_pass http://127.0.0.1:8000;
    }
}
```

## 六、日志管理

### 6.1 日志配置

日志文件位置：
- 应用日志: `logs/app.log`
- 错误日志: `logs/error.log`

### 6.2 日志轮转

使用logrotate管理日志轮转：

```bash
# /etc/logrotate.d/requirement-estimation
/logs/app.log {
    daily
    rotate 30
    compress
    delaycompress
    notifempty
    create 0640 www-data www-data
    sharedscripts
    postrotate
        systemctl reload requirement-estimation > /dev/null 2>&1 || true
    endscript
}
```

## 八、备份策略

### 8.1 数据备份

需要备份的数据：
- 上传的文档文件（`uploads/`）
- 生成的报告文件（`reports/`）
- 应用日志（`logs/`）

### 8.2 备份脚本

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="/backup/requirement-estimation"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR/$DATE

# 备份数据
cp -r uploads $BACKUP_DIR/$DATE/
cp -r reports $BACKUP_DIR/$DATE/
cp -r logs $BACKUP_DIR/$DATE/

# 压缩
tar -czf $BACKUP_DIR/backup_$DATE.tar.gz -C $BACKUP_DIR $DATE
rm -rf $BACKUP_DIR/$DATE

# 删除30天前的备份
find $BACKUP_DIR -name "backup_*.tar.gz" -mtime +30 -delete
```

## 九、故障排查

### 9.1 常见问题

#### 问题1：服务启动失败

**症状**：启动时报错 "Port 8000 already in use"

**解决**：
```bash
# 查找占用端口的进程
lsof -i:8000
# 或
netstat -tunlp | grep 8000

# 杀死进程
kill -9 <PID>
```

#### 问题2：LLM调用失败

**症状**：日志显示 "LLM request failed"

**解决**：
1. 检查API Key是否正确
2. 检查网络连接
3. 检查API额度是否用完
4. 查看详细错误日志

#### 问题3：文件解析失败

**症状**：上传文档后提示解析失败

**解决**：
1. 检查文档格式是否为.docx
2. 检查文档是否包含"需求内容说明"章节
3. 查看后端错误日志

## 十、升级维护

### 10.1 版本升级

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 更新依赖
cd backend
pip install -r requirements.txt --upgrade

# 3. 重启服务
sudo systemctl restart requirement-estimation
```

### 10.2 回滚策略

如果升级出现问题，快速回滚：

```bash
# 1. 切换到上一个版本
git checkout <previous-version-tag>

# 2. 重启服务
sudo systemctl restart requirement-estimation
```
