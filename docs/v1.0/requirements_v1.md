## 1. 概述

### 1.1 目的与范围

本文档用于定义“需求评估系统”的初始需求（以仓库当前实现能力为准），用于产品/研发/测试/运维/集成方对齐，并作为后续迭代的需求基线。

**In Scope（包含范围）**
- 角色化入口：管理员（admin）/项目经理（manager）/专家（expert）
- 任务主链路：上传需求→AI处理→人机协作修正→提交/分配→多轮专家评估→报告版本→归档
- 采购严肃增强（可审计/可追溯）：
  - A层证据库：材料导入、切块检索、在线预览（不提供下载）
  - B层系统画像：草稿/发布、字段可回链证据
  - I层 ESB 索引：明细导入/检索/统计（仅集成点提示，不扩范围）
  - C层 代码扫描：Spring Boot/Java 轻量扫描产出能力目录（事实层，不扩范围）
  - 证据等级（E0~E3）规则配置与任务内覆盖修正留痕
  - 区间估算 + 置信度 + 假设 + 风险提示
- 报告与指标：
  - 多轮评估报告（PDF）与版本管理
  - AI效果报告（指标快照 + 聚合），专家视图匿名化
- 基础配置管理：系统清单、COSMIC规则、用户/专家库字段、部门字典、通知中心、个人中心
- 对外集成：提供 RESTful API；支持 DevOps/外部系统调用评估接口（可选回调）

**Out of Scope（不包含范围）**
- 端到端自动报价/自动招采定价（仅提供评估与风控提示）
- 全量代码语义理解与自动扩范围（任何召回信号仅用于“提示复核”，不得自动扩大需求范围）
- 扫描版 PDF OCR（本期仅保证可抽取文本的 PDF）
- 生产级数据库/缓存/对象存储/K8s/CI-CD 的完整落地（当前以文件存储为主，可平滑演进）

### 1.2 背景、约束与关键假设

**现状与痛点**
- 人工拆分效率低、耗时长，评估标准不统一、偏差大
- 过程不可追溯：系统归属/拆分依据不清，难复盘复用
- 采购/投标场景中，“漏估/扩范围”带来成本与合规风险

**约束（技术/资源/时间/合规）**
- 当前存储以文件（`data/*.json`）为主：适合单机/小规模，天然限制多实例强一致与高并发写入
- AI/Embedding 依赖外部模型服务（DashScope）：Key 未配置时系统可启动，但 AI 处理可能失败或走降级策略（以部署配置为准）
- 证据材料对专家仅支持在线预览，不提供下载

**关键假设（含可验证方式）**
- A1：需求输入以 DOCX 为主，且能抽取到“需求正文”（可通过上传样例验证）
- A2：企业存在系统台账/系统清单（至少系统名称/简称），可用于系统识别的候选与校验（可通过系统清单配置导入验证）
- A3：采购严肃场景下，可提供至少一种事实信号（证据/画像/ESB/代码扫描）用于提升证据等级，否则系统需阻断并输出澄清问题（可通过无材料任务验证阻断提示）

### 1.3 术语与口径

**角色**
- admin：系统配置、用户管理、查看全部任务、分配专家、配置证据等级规则
- manager：发起任务、编辑功能点/系统、提交任务、维护证据/画像/ESB/代码扫描等索引数据、查看报告与效果指标
- expert：参与评估（多轮盲评），可在线预览与任务关联的证据材料（不下载）

**系统识别分层**
- `selected_systems`：当前证据足以“确定涉及”的系统（默认仅 E2/E3 进入）
- `maybe_systems`：证据不足或存在冲突的候选系统
- `questions`：最小澄清问题集（默认 3~8 条；按优先级截断）

**证据等级（E0~E3，默认规则可配置）**
- E3：具备“代码扫描 +（证据或画像）”等强证据组合
- E2：具备“证据/画像”或“代码/ESB”等中等证据组合；允许进入 selected
- E1：证据较弱（仅证据/画像/ESB 之一）；默认进入 maybe
- E0：无可用证据；默认进入 maybe 并强提示高风险

**偏离度口径**
- 偏离度 = `|专家均值 - AI预估| / AI预估 × 100%`（AI预估为 0 时偏离度记为 0）
- 高偏离阈值：默认 20%，用于触发下一轮评估与高偏离列表

**编号规则（本文档内）**
- 场景：`SCN-001` 起
- 功能项：`FUNC-001` 起
- 接口：`API-001` 起（详见 `docs/接口文档.md`）

**ID 前缀规则**：详见 `.claude/STRUCTURE.md` 统一定义。

## 2. 业务场景说明

### 2.1 角色与对象

**角色**
- 管理员（admin）
- 项目经理（manager）
- 专家（expert）
- 外部系统：DevOps/评审系统（可选集成方）

**核心对象**
- 任务 Task：一次评估的载体（需求正文、系统识别分层、拆分结果、评估轮次、报告版本、审计留痕）
- 功能点 Feature：按系统拆分的最小可估算单元（输入/输出/依赖、功能类型、区间估算、置信度、假设、风险）
- 证据材料 EvidenceDoc：DOCX/PPTX/PDF 原件 + 切块索引（A层）
- 系统画像 SystemProfile：结构化边界摘要（B层）
- ESB 条目 EsbEntry：集成点明细（I层）
- 代码扫描作业 CodeScanJob/结果：能力目录（C层）
- 报告版本 ReportVersion：每轮生成的 PDF（以及可选的 Excel 产物）
- 通知 Notification：站内通知（未读数、已读状态）

### 2.3 场景明细 [包含2.2中所有场景]

#### SCN-001：首次访问与登录
**场景分类**：CAT-A（账号与权限）  
**主要角色**：admin / manager / expert  
**相关对象**：User、JWT  
**关联功能项ID**：FUNC-001  
**前置条件**：
- 系统已部署可访问（前端可打开，后端健康检查可用）
**触发条件**：
- 用户访问系统入口并提交账号密码
**流程步骤**：
1. 用户访问登录页
2. 输入用户名密码并登录
3. 系统返回 JWT 并进入角色首页（任务列表/个人中心）
**输出产物**：
- 登录态（JWT）、当前用户信息（roles）
**异常与边界处理**：
- 用户名/密码错误返回 401，并提示可理解错误信息

#### SCN-002：账号治理与安全
**场景分类**：CAT-A（账号与权限）  
**主要角色**：admin / 全角色  
**相关对象**：User、Role、Department、ExpertProfile（字段）  
**关联功能项ID**：FUNC-002、FUNC-028、FUNC-030  
**前置条件**：
- admin 已登录；或用户已登录进入个人中心
**触发条件**：
- admin 维护用户/启停用/批量导入；用户修改密码/资料
**流程步骤**：
1. admin 在用户管理中新增/编辑/禁用用户与角色
2. 用户在个人中心修改资料/头像/密码
**输出产物**：
- 用户数据更新、活动日志/审计留痕（如适用）
**异常与边界处理**：
- 未授权访问返回 401/403；禁用账号无法登录

#### SCN-003：创建任务（上传需求）
**场景分类**：CAT-B（任务评估主链路）  
**主要角色**：manager  
**相关对象**：Task、RequirementDoc、AIResult（systems_data/ai_system_analysis）  
**关联功能项ID**：FUNC-003  
**前置条件**：
- manager 已登录
- 上传 DOCX 文件大小≤10MB
**触发条件**：
- 在“上传需求/新建评估”提交 DOCX
**流程步骤**：
1. 上传 DOCX 并创建任务
2. 后台解析需求正文并触发 AI 编排（系统识别→拆分→区间估算）
3. 任务进入草稿（draft），manager 进入编辑页复核
**输出产物**：
- 任务记录、AI识别分层（selected/maybe/questions）、拆分结果（按系统）
**异常与边界处理**：
- 文件类型/大小不合法返回 400/413
- 文档无法解析需求正文时，任务失败并提示原因（见 SCN-004）

#### SCN-004：AI失败与降级
**场景分类**：CAT-F（异常与运维排障）  
**主要角色**：manager / admin  
**相关对象**：Task、错误信息、日志  
**关联功能项ID**：FUNC-004、FUNC-024  
**前置条件**：
- 已创建任务并触发 AI 处理
**触发条件**：
- LLM/Embedding 失败、超时、Key 缺失、解析异常等
**流程步骤**：
1. 系统将任务标记为 failed，并记录错误信息
2. 前端提示失败原因，并允许用户重试或进入人工处理路径（以产品实现为准）
**输出产物**：
- 可理解的失败原因、任务状态与日志
**异常与边界处理**：
- Key 未配置时需明确提示“AI能力不可用/需配置”

#### SCN-005：系统识别复核
**场景分类**：CAT-B（任务评估主链路）  
**主要角色**：manager  
**相关对象**：ai_system_analysis（selected/maybe/questions）、证据引用  
**关联功能项ID**：FUNC-005、FUNC-021  
**前置条件**：
- 任务 AI 处理完成并进入草稿
**触发条件**：
- manager 打开编辑页复核系统识别分层
**流程步骤**：
1. 查看 selected/maybe/questions（含证据等级与证据摘要）
2. 根据澄清结论补充材料/画像/ESB/代码扫描，提升证据等级
3. 必要时新增系统Tab或删除不相关系统
**输出产物**：
- 系统识别分层定稿（为拆分/评估提供边界）
**异常与边界处理**：
- 证据不足时必须提示高风险，不得输出确定结论（严肃模式）

#### SCN-006：新增系统Tab自动拆分
**场景分类**：CAT-B（任务评估主链路）  
**主要角色**：manager  
**相关对象**：Task.systems_data  
**关联功能项ID**：FUNC-006  
**前置条件**：
- 任务处于草稿可编辑状态
**触发条件**：
- manager 从候选系统一键新增，或手工新增系统
**流程步骤**：
1. 新增系统Tab
2. 系统默认触发该系统的自动拆分
3. 若拆分失败，仍保留空Tab并提示原因
**输出产物**：
- 新系统的功能点列表（或空结果兜底）
**异常与边界处理**：
- 无需求正文时禁止自动拆分并提示（可重新上传/重新评估）

#### SCN-007：单系统重拆（rebreakdown）
**场景分类**：CAT-B（任务评估主链路）  
**主要角色**：manager  
**相关对象**：Task.systems_data、修改记录  
**关联功能项ID**：FUNC-007、FUNC-008  
**前置条件**：
- 任务处于草稿可编辑状态
**触发条件**：
- manager 对某系统拆分质量不满意，触发重拆
**流程步骤**：
1. 对指定系统发起重拆
2. 成功则覆盖该系统下全部功能点；失败则保留原结果
3. 写入修改记录（operation=rebreakdown_system）
**输出产物**：
- 覆盖后的拆分结果与变更留痕
**异常与边界处理**：
- 失败必须提示原因且不破坏原数据

#### SCN-008：功能点编辑（增删改）与审计
**场景分类**：CAT-B（任务评估主链路）  
**主要角色**：manager  
**相关对象**：Feature、modifications  
**关联功能项ID**：FUNC-008  
**前置条件**：
- 任务处于草稿可编辑状态
**触发条件**：
- manager 在编辑页对功能点/系统进行操作
**流程步骤**：
1. 新增/修改/删除功能点（名称、描述、输入/输出/依赖等）
2. 记录修改历史（可回放/用于指标）
3. 必要时重命名/删除系统Tab
**输出产物**：
- 更新后的功能点结构、修改记录、（可选）后续报告版本
**异常与边界处理**：
- 已提交/已评估/已归档任务不可再编辑（需返回明确提示）

#### SCN-009：提交给管理员分配专家
**场景分类**：CAT-B（任务评估主链路）  
**主要角色**：manager  
**相关对象**：Task.workflow_status、通知  
**关联功能项ID**：FUNC-009、FUNC-023  
**前置条件**：
- 草稿已复核完成
**触发条件**：
- manager 点击“提交给管理员”
**流程步骤**：
1. 任务状态从 draft → awaiting_assignment
2. 向 admin 发送“任务待分配”通知
**输出产物**：
- 任务进入待分配状态、通知记录
**异常与边界处理**：
- 非草稿状态不可重复提交

#### SCN-010：分配专家与校验
**场景分类**：CAT-B（任务评估主链路）  
**主要角色**：admin  
**相关对象**：ExpertAssignment、InviteToken、通知  
**关联功能项ID**：FUNC-010、FUNC-023、FUNC-028  
**前置条件**：
- 任务处于 awaiting_assignment
- 系统已存在可用 expert 账号
**触发条件**：
- admin 在任务详情选择 3 位专家并分配
**流程步骤**：
1. 校验：必须 3 位、不可重复、角色为 expert、账号启用、在岗
2. 为每位专家生成邀请 token 与邀请链接
3. 任务进入 evaluating，并向专家发送通知
**输出产物**：
- 分配记录、邀请链接、任务进入评估中
**异常与边界处理**：
- 校验失败返回 400 并给出可理解原因（如“专家休假不可分配”）

#### SCN-011：专家评估（草稿/提交/撤回）
**场景分类**：CAT-B（任务评估主链路）

#### SCN-012：多轮评估触发（偏离度阈值）
**场景分类**：CAT-B（任务评估主链路）  
**主要角色**：expert / admin  
**相关对象**：Deviation、HighDeviationList、current_round  
**关联功能项ID**：FUNC-012、FUNC-013、FUNC-022  
**前置条件**：
- 本轮 3 位专家均已提交
**触发条件**：
- 系统计算偏离度并检测>20%
**流程步骤**：
1. 计算每个功能点专家均值与偏离度
2. 若存在高偏离且轮次<最大轮次，则进入下一轮，仅下发高偏离功能点
3. 若无高偏离或已达最大轮次，则任务完成并固化最终专家均值
**输出产物**：
- 偏离度数据、高偏离清单、下一轮评估或最终完成状态
**异常与边界处理**：
- AI预估为 0 的偏离度需有兜底（避免除零）

### 3.2 功能项明细 [包含3.1中所有功能项]

#### FUNC-001：登录与会话
**目标/价值**：建立安全的登录态（JWT），为后续角色化功能提供统一入口。  
**入口/触发**：登录页；`POST /api/v1/auth/login`、`GET /api/v1/auth/me`。  
**前置条件**：用户账号存在且启用。  
**主流程**：
1. 用户提交用户名密码
2. 后端校验通过后签发 JWT，并返回用户基本信息与 roles
3. 前端保存 token，并按角色跳转首页
**输入/输出**：
- 输入：username、password
- 输出：token、user（id/username/displayName/roles/avatar）
**业务规则**：
- 禁用账号禁止登录
**异常与边界**：
- 用户名/密码错误返回 401
**验收标准（GWT，必须可测试）**：
- [ ] Given 用户名密码正确且账号启用，When 调用登录接口，Then 返回 JWT 且包含 roles
- [ ] Given 用户名或密码错误，When 调用登录接口，Then 返回 401 且提示“用户名或密码错误”
- [ ] Given 未携带 JWT，When 调用受保护接口，Then 返回 401 并跳转登录页
**关联**：SCN-001

#### FUNC-002：账号治理与安全
**目标/价值**：支持账号生命周期管理与基础安全能力（改密/启停用/角色）。  
**入口/触发**：用户管理页（admin）；个人中心（全角色）。  
**前置条件**：admin 已登录；或用户已登录。  
**主流程**：
1. admin 管理用户：增删改查、启用/禁用、分配角色
2. 用户自助：修改密码、维护基础资料与头像
**输入/输出**：
- 输入：用户信息、角色、状态；oldPassword/newPassword；头像文件
- 输出：用户变更结果、活动日志（如适用）
**业务规则**：
- admin 不可被禁用导致系统无法治理（建议运维约束；实现可通过流程控制）
**异常与边界**：
- 非 admin 访问用户管理接口返回 403/401
**验收标准（GWT，必须可测试）**：
- [ ] Given admin 已登录，When 禁用某用户，Then 该用户后续登录返回 403
- [ ] Given 用户已登录，When 修改密码成功，Then 旧密码不可再登录且新密码可登录
**关联**：SCN-002、SCN-028、SCN-030

#### FUNC-003：创建任务（上传需求）
**目标/价值**：将 DOCX 需求文档转为可编辑的评估任务（草稿）。  
**入口/触发**：上传需求页；`POST /api/v1/tasks`。  
**前置条件**：manager 已登录；文件为 DOCX 且≤10MB。  
**主流程**：
1. 上传 DOCX 创建任务
2. 后台解析需求正文并触发 AI 编排处理
3. 任务完成后进入草稿，可在编辑页复核
**输入/输出**：
- 输入：DOCX 文件、可选任务名称/说明
- 输出：taskId、aiStatus/progress/message
**业务规则**：
- 仅 manager 可创建任务
**异常与边界**：
- 文件格式非 DOCX 返回 400；文件过大返回 413
**验收标准（GWT，必须可测试）**：
- [ ] Given manager 已登录，When 上传合法 DOCX，Then 返回 taskId 且任务可在“我发起的任务”看到
- [ ] Given 非 manager 用户，When 调用创建任务接口，Then 返回 403
**关联**：SCN-003

#### FUNC-004：AI失败与降级
**目标/价值**：AI 失败时不“黑盒”，给出可理解原因并提供可继续推进路径。  
**入口/触发**：任务列表/进度接口；AI 处理后台。  
**前置条件**：任务已创建。  
**主流程**：
1. AI 失败时写入任务 error/message
2. 前端展示错误并允许用户重试/人工处理（产品策略）
**输入/输出**：
- 输入：任务ID
- 输出：aiStatus=failed、error/message
**业务规则**：
- 错误信息应可理解且不泄露敏感信息
**异常与边界**：
- 未配置模型 Key 时需明确提示
**验收标准（GWT，必须可测试）**：
- [ ] Given 模型 Key 缺失，When 发起 AI 任务，Then 任务进入 failed 且 message 提示需配置 Key
**关联**：SCN-004

#### FUNC-005：系统识别分层复核
**目标/价值**：以“证据约束 + 阻断”提升系统识别准确性与可审计性。  
**入口/触发**：编辑页系统校准卡片；任务结果接口。  
**前置条件**：任务 AI 初评完成（草稿）。  
**主流程**：
1. 输出 selected/maybe/questions
2. selected 默认仅包含证据等级 E2/E3 系统；其余进入 maybe
3. questions 控制在 3~8 条，面向业务可回答
**输入/输出**：
- 输入：任务ID、需求正文、证据/画像/ESB/扫描等索引信号
- 输出：ai_system_analysis（分层+证据引用+风险提示）
**业务规则**：
- 严肃模式下：证据不足禁止输出确定结论
**异常与边界**：
- 未命中证据时需提示缺失项（material/profile/code/esb）
**验收标准（GWT，必须可测试）**：
- [ ] Given 系统证据等级为 E1/E0，When 输出系统识别结果，Then 该系统进入 maybe_systems 且 questions 提示补充证据
- [ ] Given questions 超过 8 条，When 返回结果，Then 按优先级截断并返回前 3~8 条
**关联**：SCN-005

#### FUNC-006：新增系统Tab自动拆分
**目标/价值**：支持补齐系统范围并自动生成该系统功能点初稿。  
**入口/触发**：编辑页“新增系统Tab”；`POST /api/v1/requirement/systems/{task_id}`。  
**前置条件**：任务草稿可编辑且存在需求正文。  
**主流程**：
1. 新增系统Tab
2. 默认触发自动拆分；失败保留空Tab
**输入/输出**：
- 输入：task_id、system_name、（可选）system_type、auto_breakdown
- 输出：新增系统与拆分结果（或错误提示）
**业务规则**：
- 新增系统后默认进入可编辑草稿
**异常与边界**：
- 需求正文缺失时禁止自动拆分并提示
**验收标准（GWT，必须可测试）**：
- [ ] Given 任务草稿且需求正文存在，When 新增系统Tab，Then 生成该系统功能点列表（或返回空Tab并提示失败原因）
**关联**：SCN-006

#### FUNC-007：单系统重拆（rebreakdown）
**目标/价值**：允许对单系统拆分质量进行定向修复，避免全量重跑。  
**入口/触发**：编辑页“重新拆分”；`POST /api/v1/requirement/systems/{task_id}/{system_name}/rebreakdown`。  
**前置条件**：任务草稿可编辑。  
**主流程**：
1. 触发重拆并覆盖该系统功能点
2. 失败则保留原结果并提示原因
3. 写入修改记录
**输入/输出**：
- 输入：task_id、system_name
- 输出：覆盖后的功能点、breakdown_error（如有）
**业务规则**：
- 仅覆盖当前系统，不影响其他系统
**异常与边界**：
- 重拆失败不应破坏原数据
**验收标准（GWT，必须可测试）**：
- [ ] Given 重拆失败，When 返回结果，Then 原功能点列表保持不变且返回 breakdown_error
**关联**：SCN-007

#### FUNC-008：功能点编辑与审计
**目标/价值**：支持人机协作修正，形成可回放修改记录与后续校准数据。  
**入口/触发**：编辑页；`PUT /api/v1/requirement/features/{task_id}` 等系统操作接口。  
**前置条件**：任务处于 draft 且未提交。  
**主流程**：
1. 功能点增删改（模块/名称/描述/输入/输出/依赖等）
2. 系统Tab：重命名、删除、新增、重拆
3. 每次修改写入 modifications（用于报告与指标）
**输入/输出**：
- 输入：操作类型（add/update/delete）、feature_data
- 输出：更新成功与最新 modifications
**业务规则**：
- 已确认/已提交/已归档任务不可修改
**异常与边界**：
- 非法索引/系统不存在返回 400
**验收标准（GWT，必须可测试）**：
- [ ] Given 任务为草稿，When 修改功能点业务描述，Then modifications 记录包含 operation=update 且可查询
- [ ] Given 任务已提交，When 再次修改功能点，Then 返回 400 且提示“该任务已确认/不能修改”
**关联**：SCN-008

#### FUNC-009：提交给管理员分配专家
**目标/价值**：将草稿冻结进入评估流程，确保评估口径一致。  
**入口/触发**：编辑页“提交给管理员”；`POST /api/v1/tasks/{task_id}/submit-to-admin`。  
**前置条件**：任务处于 draft，manager 对任务有访问权限。  
**主流程**：
1. draft → awaiting_assignment
2. 发送通知给 admin
**输入/输出**：
- 输入：task_id
- 输出：提交成功
**业务规则**：
- 非草稿状态不可提交
**异常与边界**：
- 任务不存在返回 404
**验收标准（GWT，必须可测试）**：
- [ ] Given 任务为草稿，When 提交给管理员，Then 任务状态变更为 awaiting_assignment 且 admin 收到通知
**关联**：SCN-009

#### FUNC-010：分配专家与校验
**目标/价值**：保证评估由“合规专家账号”参与，避免角色/状态错误导致流程中断。  
**入口/触发**：任务详情页分配；`POST /api/v1/tasks/{task_id}/assign-experts`。  
**前置条件**：admin 已登录；任务处于 awaiting_assignment。  
**主流程**：
1. 选择 3 位专家
2. 校验账号：存在、角色为 expert、启用、在岗
3. 生成邀请 token 与链接，任务进入 evaluating
**输入/输出**：
- 输入：expertIds（3个）
- 输出：inviteLinks（含 token）
**业务规则**：
- 必须分配 3 位且不可重复
**异常与边界**：
- 专家休假/禁用应拒绝分配
**验收标准（GWT，必须可测试）**：
- [ ] Given expertIds 数量不为 3，When 分配专家，Then 返回 400 且提示“必须分配3位专家”
- [ ] Given 专家 on_duty=false，When 分配该专家，Then 返回 400 且提示“专家当前休假”
**关联**：SCN-010

#### FUNC-011：专家评估（草稿/提交/撤回）
**目标/价值**：支持专家按轮盲评，降低锚定效应并保证过程可追溯。  
**入口/触发**：评估页；`GET/POST /api/v1/evaluation/{task_id}`。  
**前置条件**：expert 已分配且 token 有效；任务处于 evaluating。  
**主流程**：
1. 拉取评估数据（按轮次过滤功能点）
2. 单元格内编辑人天，自动保存草稿
3. 提交评估；在规则允许下可撤回
**输入/输出**：
- 输入：token、round、evaluations（feature_id→days）
- 输出：提交时间、草稿保存成功
**业务规则**：
- 未填写的值提交时默认采用 AI 预估值
**异常与边界**：
- token 无效返回 403；轮次不匹配返回 400
**验收标准（GWT，必须可测试）**：
- [ ] Given 专家已分配，When 拉取评估数据，Then 返回仅本轮功能点并包含 aiEstimatedDays
- [ ] Given 专家未提供某功能点值，When 提交评估，Then 该功能点按 AI 预估值落库
**关联**：SCN-011

#### FUNC-012：多轮评估触发（偏离度阈值）
**目标/价值**：聚焦争议点，提高收敛效率与输出稳定性。  
**入口/触发**：系统在本轮 3 位专家均提交后自动触发。  
**前置条件**：本轮评估完成。  
**主流程**：
1. 计算专家均值与偏离度
2. 偏离度>20% 的功能点进入高偏离清单
3. 若存在高偏离且未达最大轮次，则进入下一轮，仅下发高偏离功能点
4. 否则任务完成并固化 expert_final
**输入/输出**：
- 输入：evaluations、aiEstimatedDays
- 输出：deviations、highDeviationFeatures、currentRound 更新
**业务规则**：
- 最大轮次默认 3，可配置
**异常与边界**：
- AI 预估为 0 的偏离度需兜底为 0
**验收标准（GWT，必须可测试）**：
- [ ] Given 存在偏离度>20%的功能点且 round<maxRounds，When 本轮结束，Then 系统进入下一轮且仅下发高偏离功能点
- [ ] Given 无高偏离功能点，When 本轮结束，Then 任务状态变更为 completed 并写入 expert_final
**关联**：SCN-012

#### FUNC-013：报告生成与版本管理
**目标/价值**：输出可审计报告，支持按轮追溯与下载。  
**入口/触发**：系统在轮次结束时自动生成；报告下载接口。  
**前置条件**：任务存在且用户具备访问权限（admin/manager）。  
**主流程**：
1. 生成 PDF 报告并记录 report_versions
2. 支持下载最新报告与指定版本
**输入/输出**：
- 输入：task_id、report_id
- 输出：PDF 文件流
**业务规则**：
- 文件访问需做路径校验，防止路径遍历
**异常与边界**：
- 报告不存在返回 404
**验收标准（GWT，必须可测试）**：
- [ ] Given 任务已有报告版本，When 下载最新报告，Then 返回 PDF 文件
- [ ] Given 非法文件路径，When 请求下载，Then 返回 403
**关联**：SCN-013

#### FUNC-014：归档/删除/保留策略
**目标/价值**：控制数据生命周期，避免无边界增长并满足审计/复盘需求。  
**入口/触发**：任务列表操作；系统清理逻辑。  
**前置条件**：用户具备权限。  
**主流程**：
1. 归档任务（archived）
2. 删除限制：仅 draft/awaiting_assignment 可删
3. 清理策略：完成/失败任务超过保留天数自动清理（默认 7 天，可配置）
**输入/输出**：
- 输入：task_id
- 输出：归档/删除结果；清理数量
**业务规则**：
- 评估中/已完成任务不可删除（需归档）
**异常与边界**：
- 状态不允许操作返回 400
**验收标准（GWT，必须可测试）**：
- [ ] Given 任务为 evaluating，When 请求删除，Then 返回 400 且提示“当前状态不可删除”
**关联**：SCN-014

#### FUNC-015：证据库（A层）
**目标/价值**：为系统识别/拆分提供可检索、可回放的证据依据。  
**入口/触发**：证据库导入/检索接口；知识库页面。  
**前置条件**：manager 已登录；选择 system_name。  
**主流程**：
1. 导入 DOCX/PPTX/PDF（≤50MB），抽取文本并切块入库
2. 按系统与查询文本检索证据块，返回相似度与片段信息
**输入/输出**：
- 输入：文件、system_name、trust_level/doc_date 等
- 输出：doc_id、chunk_count、检索结果列表
**业务规则**：
- trust_level 仅支持 高/中/低
**异常与边界**：
- 不支持格式返回 400；文件过大返回 413
**验收标准（GWT，必须可测试）**：
- [ ] Given manager 导入 PDF 证据，When 文件可抽取文本，Then 返回 chunk_count>0 且可检索到相似结果
**关联**：SCN-015

#### FUNC-016：证据预览权限（不下载）
**目标/价值**：在保证材料安全前提下，让专家可验证证据原文。  
**入口/触发**：评估页/详情页“预览证据”；`GET /api/v1/knowledge/evidence/preview/{doc_id}`。  
**前置条件**：证据已关联到任务；expert 参与该任务。  
**主流程**：
1. 校验权限（expert 必须参与任务且 doc_id 在关联范围内）
2. PDF 返回 inline 预览；其它类型返回文本预览（可截断）
**输入/输出**：
- 输入：doc_id、（可选）task_id
- 输出：inline PDF 或文本预览 JSON
**业务规则**：
- 不提供下载链接/附件下载
**异常与边界**：

### 4.2 非功能需求明细 [包含4.1中所有非功能需求]

#### NFR-101：鉴权与权限边界
**需求分类**：安全  
**适用范围**：全局 API（除公开接口如登录/健康检查）  
**指标与口径**：
- 未登录访问受保护接口返回 401
- 越权访问返回 403（含 expert 访问非任务范围资源）
**验收方法/证据**：
- 接口冒烟测试与权限用例（覆盖 admin/manager/expert）
- 日志可定位 401/403 触发点（含关键字段）
**异常与边界**：
- 管理接口如启用 `X-API-Key`，需明确其适用范围与缺失时的行为
**关联**：SCN-001、SCN-024；`/api/v1/auth/*`、核心业务接口（详见 `docs/接口文档.md`）

#### NFR-102：证据预览与数据安全
**需求分类**：安全  
**适用范围**：证据列表/检索/预览接口与前端预览入口  
**指标与口径**：
- expert 仅能预览其参与任务关联证据；不提供下载能力
**验收方法/证据**：
- 越权访问测试：非参与任务的证据预览返回 403
- 前端无下载入口；后端无直接下载接口暴露（以接口文档为准）
**异常与边界**：
- 预览渲染失败时允许降级为文本预览，但仍需遵守权限边界
**关联**：SCN-016；证据相关接口（详见 `docs/接口文档.md`）

#### NFR-103：SSRF/路径遍历/上传安全
**需求分类**：安全  
**适用范围**：外部回调、报告/文件路径、上传接口  
**指标与口径**：
- callback_url 仅允许内网/本机/私有网段（防 SSRF）
- 下载/读取路径需校验在允许目录内（防路径遍历）
- 上传文件限制类型与大小；文件名清洗
**验收方法/证据**：
- SSRF 用例：callback_url 为公网域名返回 400
- 路径遍历用例：非法路径不可访问
- 上传用例：超限/不支持类型返回可理解错误
**异常与边界**：
- 需要记录拦截原因，便于排障与审计
**关联**：SCN-031；上传/回调/报告相关接口（详见 `docs/接口文档.md`）

#### NFR-201：文件存储一致性与清理
**需求分类**：可用性/可靠性  
**适用范围**：`data/` 下任务/配置/索引等文件写入与清理机制  
**指标与口径**：
- 写入采用“文件锁 + tmp + replace”原子替换
- 已完成/失败任务按 `TASK_RETENTION_DAYS` 清理；活跃任务不自动清理
**验收方法/证据**：
- 并发写入/重启恢复测试（数据不损坏、可读取）
- 清理脚本/任务验证（仅清理满足条件的数据）
**异常与边界**：
- 文件权限/磁盘满等问题需可观测并给出可理解提示
**关联**：SCN-014、SCN-025

#### NFR-301：健康检查与关键日志
**需求分类**：可观测性  
**适用范围**：服务启动、核心链路、外部依赖调用  
**指标与口径**：
- 健康检查：`GET /api/v1/health`
- 日志覆盖关键事件：任务创建/失败原因/评估提交/规则变更/回调失败等
**验收方法/证据**：
- 部署后可复现的健康检查命令与返回示例
- 故障注入后可通过日志快速定位（如模型不可用、文件写入失败）
**异常与边界**：
- 不记录真实 Secret；敏感信息脱敏
**关联**：SCN-025

#### NFR-401：单机可用与扩展路径
**需求分类**：性能/容量  
**适用范围**：任务主链路、检索与索引  
**指标与口径**：
- 单机小规模可用（文件存储 + 本地向量检索）
- 数据量增长后支持演进：Milvus/DB/对象存储/异步作业（路径在设计中明确）
**验收方法/证据**：
- 压测报告/监控数据/演练记录（按项目实际补齐指标）
**异常与边界**：
- AI 调用需有超时/重试与降级口径（以配置为准）
**关联**：SCN-003~SCN-014

#### NFR-501：文件格式与 API 版本
**需求分类**：兼容性  
**适用范围**：上传/导入/解析与 API 对外集成  
**指标与口径**：
- 支持 DOCX（需求/画像/证据）、PPTX（画像/证据）、PDF（证据）、CSV/XLSX（ESB/导入模板）
- API 前缀 `/api/v1`；版本策略需明确
**验收方法/证据**：
- 样例文件上传/导入测试；错误提示清晰
- 集成方按版本策略进行调用与升级
**异常与边界**：
- 扫描版 PDF（无法抽取文本）应明确提示并拒绝入库（本期不做 OCR）
**关联**：SCN-003、SCN-015、SCN-018、SCN-026

### 4.3 安全（参考维度）
- 认证与授权：JWT + 角色校验（admin/manager/expert）；敏感管理接口额外支持 `X-API-Key`（生产环境建议仅保留 JWT 方案）
- 证据预览权限：expert 仅能预览其参与任务关联证据；不提供下载
- SSRF 防护：外部回调地址仅允许内网/本机/私有网段
- 路径遍历防护：下载报告/文档等需校验路径在允许目录内
- 文件上传安全：限制扩展名与大小；文件名清洗
- 审计与追踪：证据等级规则变更、任务内覆盖修正需写入审计日志

### 4.4 性能与容量（参考维度）
- 单机小规模可用：文件存储 + 全量扫描向量检索（数据量增长后需迁移 Milvus/DB）
- 指标口径：支持按任务/系统/模块聚合；允许按时间区间过滤
- 超时/重试：AI 调用存在超时与重试；回调请求有连接与总超时

### 4.5 可用性与可靠性（参考维度）
- 文件写入一致性：文件锁 + 临时文件写入 + 原子替换（`tmp + replace`）
- 任务保留策略：完成/失败任务按保留天数清理（可配置）；活跃任务不自动清理
- 降级策略：
  - Embedding 可在允许条件下启用本地降级（确定性哈希）
  - LLM Key 缺失时应给出明确提示，主链路可能失败但系统可登录与配置

### 4.6 可观测性（参考维度）
- 健康检查：`GET /api/v1/health`
- 日志：记录关键事件（任务创建/失败原因/评估提交/规则变更等）
- 指标：AI效果报告快照（可用于趋势与告警）

### 4.7 兼容性（参考维度）
- 文件格式：DOCX（需求/画像/证据）、PPTX（画像/证据）、PDF（证据）、CSV/XLSX（ESB/导入模板）
- API：统一前缀 `/api/v1`；历史接口（`/requirement/*`）与新任务接口（`/tasks/*`）并存，需明确弃用路线

### 4.8 交互与体验（参考维度，可选）
- 关键路径交互：表单校验、空状态、加载态、错误态
- 可用性：学习成本、任务完成率、误操作防护
- 可访问性（A11y，如适用）

## 5. 权限与合规

### 5.1 权限矩阵（关键能力）

| 角色 | 操作/权限 | 资源范围 | 备注 |
|---|---|---|---|
| admin | 查看全部任务/分配专家/配置系统清单/用户管理/证据等级规则 | 全局 | admin 不可兼职其他角色（产品约束） |
| manager | 创建任务/编辑草稿/提交任务/维护证据A与画像B/ESB/代码扫描/查看报告与指标 | 自己发起的任务 + 配置项 | 可兼任 expert |
| expert | 查看“我参与的评估”/评估提交与撤回/预览任务关联证据/查看匿名效果指标 | 参与的任务范围 | 预览不提供下载 |

### 5.2 客户信息与合规

**可能涉及的敏感字段（需按企业合规要求处理）**

### 6.2 错误码与提示语（示例）

| 错误码 | 场景 | 提示语 | HTTP状态码 | 处理建议 |
|---|---|---|---|---|
| AUTH_MISSING | 未登录访问受保护接口 | 缺少Authorization | 401 | 重新登录 |
| AUTH_FORBIDDEN | 角色不匹配 | 权限不足 | 403 | 切换账号/联系管理员 |
| ADMIN_KEY_MISSING | 管理接口缺少配置 | ADMIN_API_KEY 未配置 | 500 | 配置 `ADMIN_API_KEY` 或使用 admin JWT |
| FILE_TOO_LARGE | 上传文件过大 | 文件过大，最大允许XXMB | 413 | 压缩/拆分文件 |
| INVALID_FILE_TYPE | 文件类型不支持 | 仅支持 DOCX/PPTX/PDF | 400 | 更换文件格式 |

### 6.3 指标与计算口径

**偏离度（Deviation）**
- `deviation = |mean(expert_days) - ai_days| / ai_days × 100%`
- 高偏离阈值：默认 20%

**AI效果报告关键指标（示例口径）**
- 人天准确率：`1 - 人天字段被修改次数 / 初始功能点数`
- 功能点保留率：`当前功能点数 / 初始功能点数`
- 字段修改率：`描述/输入/输出/依赖项被修改次数 / 初始功能点数`
- 知识库命中率：按任务/阶段记录“检索命中”比例（以实现为准）
- 系统识别准确率：`当前系统数 / 初始系统数`
