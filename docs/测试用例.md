### 测试用例1：基础需求评估流程

#### 1.1 测试目标
验证完整的需求评估流程（上传→处理→下载报告）

#### 1.2 测试步骤

```bash
# 1. 健康检查
curl http://localhost:443/api/v1/health

# 2. 上传需求文档
curl -X POST "http://localhost:443/api/v1/requirement/upload" \
  -F "file=@tests/fixtures/反洗钱系统需求.docx"

# 3. 获取task_id（从响应中提取）
TASK_ID="<提取的task_id>"

# 4. 轮询查询状态
while true; do
  STATUS=$(curl -s "http://localhost:443/api/v1/requirement/status/$TASK_ID" | jq -r '.data.status')
  echo "当前状态: $STATUS"

  if [ "$STATUS" = "completed" ]; then
    echo "评估完成"
    break
  elif [ "$STATUS" = "failed" ]; then
    echo "评估失败"
    exit 1
  fi

  sleep 5
done

# 5. 下载报告
curl -X GET "http://localhost:443/api/v1/requirement/report/$TASK_ID" \
  -o "评估报告.xlsx"
```

#### 1.3 验证点

- [ ] 健康检查返回200
- [ ] 上传成功，返回task_id
- [ ] 任务状态最终变为completed
- [ ] Excel报告成功下载
- [ ] 报告包含汇总统计sheet
- [ ] 报告包含各系统功能点sheet

### 测试用例2：人机协作功能

#### 2.1 测试目标
验证AI评估结果的人工修正功能

#### 2.2 测试步骤

```python
import requests
import json

BASE_URL = "http://localhost:443"

# 1. 上传并等待完成
def test_upload_and_wait():
    files = {"file": open("tests/fixtures/需求文档.docx", "rb")}
    response = requests.post(f"{BASE_URL}/api/v1/requirement/upload", files=files)
    result = response.json()
    task_id = result["data"]["task_id"]

    # 等待完成
    while True:
        status_resp = requests.get(f"{BASE_URL}/api/v1/requirement/status/{task_id}")
        status_data = status_resp.json()["data"]
        if status_data["status"] == "completed":
            break
        time.sleep(5)

    return task_id

# 2. 获取评估结果
task_id = test_upload_and_wait()
result_resp = requests.get(f"{BASE_URL}/api/v1/requirement/result/{task_id}")
systems_data = result_resp.json()["data"]

print(f"识别系统: {list(systems_data['systems_data'].keys())}")

# 3. 添加新功能点
add_data = {
    "system": list(systems_data['systems_data'].keys())[0],
    "operation": "add",
    "feature_data": {
        "序号": "1.5",
        "功能模块": "测试模块",
        "功能点": "测试功能点",
```

## 三、接口级回归测试清单（建议）

1. **管理接口鉴权**
   - 未携带 `X-API-Key` 调用 `POST /api/v1/knowledge/import`，期望 401
   - 携带正确 `X-API-Key`，期望 200

2. **任务持久化**
   - 创建任务后重启后端（或容器），再次查询 `GET /api/v1/requirement/status/{task_id}` 仍可读

3. **知识库分类导入**
   - `knowledge_type=system_profile` 导入 CSV，期望进入系统知识
   - `knowledge_type=feature_case` 导入 CSV，期望进入功能案例

4. **上传扩展名兼容**
   - 上传 `*.DOCX`（大写扩展名），期望被接受

### 测试用例8：并发处理

#### 8.1 测试目标
验证系统能否并发处理多个任务

#### 8.2 测试方法

使用Locust进行性能测试：

```python
from locust import HttpUser, task, between

class RequirementUser(HttpUser):
    wait_time = between(1, 3)

    @task
    def upload_and_check_status(self):
        # 1. 上传文件
        with open("tests/fixtures/需求文档.docx", "rb") as f:
            response = self.client.post("/api/v1/requirement/upload", files={"file": f})

        task_id = response.json()["data"]["task_id"]

        # 2. 轮询状态（最多10次）
        for _ in range(10):
            response = self.client.get(f"/api/v1/requirement/status/{task_id}")
            status = response.json()["data"]["status"]

            if status == "completed":
                return
            elif status == "failed":
                raise Exception("Task failed")

            time.sleep(5)

# 运行测试
# locust -f locustfile.py --host=http://localhost:443 --users 10 --spawn-rate 2
```

#### 8.3 预期结果

- 成功率 > 95%
- 平均响应时间 < 5秒
- 无崩溃或内存泄漏
