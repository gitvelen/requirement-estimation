# v2.7 Change Management Review

## 2026-04-08 Codex

### 审查结论
- 无阻断项，建议按同版本补丁 `CR-20260408-001` 执行。
- 本次属于已发布版本上的缺陷修复，不涉及 API 契约、数据库 schema 或新功能扩范围；补丁虽覆盖导入解析、AI 建议处理与重试链路，但均落在既有画像闭环内，审查口径维持 `diff-only`。

### 审查事实
- 用户先复现两类问题：一是需求文档/技术方案导入后画像字段常被目录标题或低相关片段污染，二是技术方案导入后 D3/D4/D5 出现建议，但点击“采纳新建议”或“忽略”后卡片不消失。
- 根因之一在导入链路：文档原始解析文本未经统一清洗直接进入画像抽取和知识摘要，目录、封面、页码行会放大为错误建议。
- 根因定位到 `backend/service/system_profile_service.py`：v2.7 平铺 canonical 建议在 accept/ignore/rollback 时被错误迁移为旧嵌套结构，导致采纳写空值、忽略写空列表。
- 另一处补强点在运行态闭环：支持的文档导入需要保存清洗后正文快照，信息展示页“重新生成 AI 建议”才能稳定重放同一份输入。
- 运行态还存在放大风险：服务启动迁移会继续污染纯平铺建议，因此补丁必须同时修动作链路和启动迁移。
- 用户已确认当前 `贷款核算` 的已生成坏建议数据可清理，不要求做全量历史批修。

### 建议进入阶段
- 进入 Deployment，部署到 STAGING/TEST 后等待业务复验。
