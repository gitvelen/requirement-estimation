## 1. 变更意图（User Story 格式）
- **作为** 系统画像使用者 **我想要** 在 v2.7 导入需求文档/技术方案后，系统能忽略目录与封面噪声、把正文内容尽量落到 D3/D4/D5 等正确画像字段，并且在信息展示页对 AI 建议执行“采纳新建议 / 忽略 / 重新生成”后立即生效 **以便于** 正常完成文档导入后的画像确认闭环，而不是只看到标题片段或持续看到“检测到 AI 建议变更”卡片不消失。

## 2. 变更点（What）
- 清洗需求文档、技术方案和知识导入文本中的目录、封面、页码等噪声，避免把章节列表直接写入画像或知识摘要。
- 补强 `document_skill_adapter` 的技术方案抽取逻辑，使正文内容可落到 D3 集成接口、D4 技术架构、D5 约束风险等多个 canonical 字段。
- 为支持的 v2.7 文档导入记录保存运行态 `input_snapshot`，使信息展示页“重新生成 AI 建议”能够基于同一份清洗后正文重放。
- 修复 `system_profile_service` 对 v2.7 平铺 canonical 建议键的读取、采纳、忽略与回滚逻辑。
- 修复启动迁移，避免服务重启时把平铺建议错误迁成旧嵌套结构。
- 为当前 `贷款核算` 画像清理受损建议状态，并回填用户已点击“采纳”的 D3/D4 字段。
- 增补后端与前端定向回归用例，覆盖文档清洗、多域建议、平铺建议采纳/忽略与重试场景。

## 4. 兼容性与迁移（如适用）
- 向后兼容：保留 legacy 嵌套建议读取路径，同时优先支持 v2.7 平铺 canonical 键。
- 数据迁移：不做全量批修；仅对 `贷款核算` 当前受损记录进行一次性清理与回填。
- 运行态兼容：仅带 `document_ingest_v1` 快照的新导入记录支持“重新生成 AI 建议”；历史无快照记录继续提示重新上传。
- 配置变更/开关/灰度：无新增配置项。

## 5. 风险与回滚
- 风险：建议兼容层同时覆盖 flat / nested / mixed shape，且导入抽取逻辑覆盖 requirements / tech_solution / knowledge 三条链路，若处理不当可能影响旧建议场景或把文档噪声写回画像。
- 监控/告警：重点核对信息展示页建议卡片是否消失、badge 是否清空、重新生成链路是否成功、后台健康检查是否正常。
- 回滚条件：导入后再次出现目录/标题噪声写入画像，采纳/忽略再次不生效，或服务重启后建议数据再次被污染。
- 回滚步骤：
  - 回退到补丁前的 backend 镜像/代码版本并重建 `requirement-backend`。
  - 恢复 `data/system_profiles.json` 中本次修复前的备份副本。

## 6. 验收与验证
### 验收标准（GWT 格式，🔴 MUST 至少一条）
- Given 技术方案正文前存在封面与目录 When 用户导入文档 Then 系统画像建议不得直接使用“目录/章节名/页码”作为字段值。
- Given 技术方案正文包含集成、架构和风险描述 When 用户导入文档 Then 系统可生成 D3 / D4 / D5 多域平铺 canonical 建议。
- Given 系统画像存在由技术方案导入生成的平铺 canonical AI 建议 When 用户在信息展示页点击“采纳新建议” Then 对应 canonical 字段更新为建议值，建议卡片消失。
- Given 系统画像存在由技术方案导入生成的平铺 canonical AI 建议 When 用户在信息展示页点击“忽略” Then 忽略值被持久化，域 badge 与建议卡片消失。
- Given 用户点击“重新生成 AI 建议”且最近一次导入包含 `document_ingest_v1` 快照 When 系统重放该快照 Then 返回新的执行记录并刷新画像建议。
- Given 后端服务重启 When 系统重新加载已有画像数据 Then 平铺 canonical AI 建议不会被错误迁移成旧嵌套空结构。
