## 1. 变更意图（User Story 格式）
- **作为** 项目经理 / 管理员 **我想要** 页面继续保持上一版已确认的交互形式，并把导入结果、状态和建议内容都转成用户可读文本 **以便于** 我可以直接理解结果并完成人工验收，而不需要理解后台 `execution_id`、对象结构或内部原因码。

## 2. 变更点（What）
- 收敛 PM「知识导入」页顶部状态、导入历史与本次结果展示，移除前端直接暴露的 `execution_id / scene / status code`。
- 修复「信息展示」页结构化建议对象直接渲染为 `[object Object]` 的问题。
- 保持左侧菜单独立滚动，避免右侧页面滚动时联动滚动。
- 收敛 admin「服务治理」与「系统清单」结果区：展示系统名称和用户可读原因，不直接展示内部 ID 与原始 reason code。
- 同步 Testing 阶段文档链与主文档，避免“代码已修复、文档仍指导错误验收”的漂移。
