# 需求文档附件解析需求

## 问题描述

项目经理在系统中发起了一个名为"qw"的评估任务，对《2026年管理会计集市系统中收模型优化等共计33个迭代优化需求 (1).docx》进行了工作量评估。

通过查看详情，发现该文档内部还有 docx、xlsx 等格式的附件，并未充分解析，也就没有拆出相应的功能点，这是重大缺失。

## 需求

实际估算工作中，提交的需求文档，可能带有 docx、xlsx、pptx、pdf 等格式的内部附件，需要一并解析。

## 用户确认的实施方案

1. **实施范围**：一次性支持所有格式（DOCX/XLSX/PDF/PPTX）
2. **报告展示**：不需要标注附件来源（简化实现）
3. **错误处理**：单个附件解析失败时继续处理其他附件
4. **性能限制**：使用默认值（递归深度3层、单附件10MB、总数20个、超时5分钟）

- 用户口头描述
- 实际场景案例（qw 评估任务）
- 问题文档：《2026年管理会计集市系统中收模型优化等共计33个迭代优化需求 (1).docx》
