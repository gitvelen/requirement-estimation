# 实现检查清单（模板）

## 实现前检查
- [ ] 已阅读相关现有代码/文档（requirements/design/plan）
- [ ] 已对齐范围与"不做什么"（避免范围漂移）
- [ ] 已明确验收标准（可判定、可复现）
- [ ] 已明确影响面：要改的模块/文件/接口/数据
- [ ] 如涉及线上行为变化：已准备开关/灰度与回滚思路
- [ ] 开发环境就绪：
  - [ ] 依赖已安装（包管理器无报错）
  - [ ] 项目可正常构建/启动
  - [ ] 数据库/外部服务已就绪（或有 mock）
  - [ ] 环境变量/配置文件已准备（参考 `.env.example` 或配置模板）

## 实现中检查
- [ ] 代码简洁清晰，无过度设计
- [ ] 遵循项目代码规范（检查项目根目录是否有 lint/format 配置文件，如无则按语言社区主流规范创建）
- [ ] 未引入非必要依赖
- [ ] 关键路径：鉴权/输入校验/错误码一致/幂等（如适用）
- [ ] 安全检查（SQL注入、XSS、SSRF、越权等）
- [ ] 可观测性：关键日志/指标/告警点（如适用）
- [ ] 数据变更：向后兼容、迁移可回滚（如适用）

## 实现后检查
- [ ] 代码可正常运行（本地/测试环境）
- [ ] 代码风格检查通过：
  - [ ] `...`（format/lint，按项目实际工具填写）
- [ ] 测试通过：
  - [ ] `...`（unit/integration/e2e，按项目实际填写）
- [ ] 对照验收标准自测并记录证据（命令输出/日志/截图）
- [ ] 文档同步更新（如有变更）：requirements/design/plan/操作说明
- [ ] 敏感信息检查：不提交/不记录 secret、生产数据、个人信息
