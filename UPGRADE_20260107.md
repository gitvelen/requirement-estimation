# Docker 容器安全加固说明

## 📋 改动概览

本次改动（2026-01-07）对 Docker 容器进行了安全加固和性能优化：

### ✅ 已完成的改动

1. **引入 uv 包管理器**
   - 比传统 pip 快 10-100 倍
   - 支持并行下载和安装
   - 完全兼容 pip 命令

2. **创建非 root 用户运行容器**
   - 用户名：appuser
   - 提升容器安全性
   - 符合安全审计要求

3. **支持私服镜像源**
   - 通过 `UV_INDEX_URL` 参数配置
   - 支持内网部署
   - 支持带认证的私服

4. **优化文件权限**
   - 所有文件使用 `--chown` 设置正确权限
   - 防止权限问题

## 🚀 使用方法

### 场景 1：使用默认配置（腾讯云镜像）

```bash
# 1. 验证配置
./verify-docker.sh

# 2. 构建镜像
docker build -t requirement-backend .

# 3. 启动服务
docker-compose up -d

# 4. 验证非 root 用户
docker exec requirement-backend whoami
# 输出应该是：appuser
```

### 场景 2：使用私服镜像（内网部署）

**步骤 1：修改 docker-compose.yml**

```yaml
services:
  backend:
    build:
      context: .
      args:
        # 修改为你的私服地址
        - UV_INDEX_URL=http://192.168.1.200:8080/simple
        # 或带认证：
        # - UV_INDEX_URL=http://admin:password@192.168.1.200:8080/simple
```

**步骤 2：重新构建**

```bash
# 清除缓存重新构建
docker-compose build --no-cache backend

# 启动服务
docker-compose up -d
```

### 场景 3：本地开发使用 uv

```bash
# 1. 安装 uv
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. 使用 uv 安装依赖
uv pip install -r requirements.txt --index-url https://mirrors.cloud.tencent.com/pypi/simple

# 3. 运行应用
python3 backend/app.py
```

## 🔒 安全性对比

### 改动前（root 用户）
```
容器被攻破 → 获得 root 权限 → 可访问主机文件系统
危险等级：⚠️ 高
```

### 改动后（非 root 用户）
```
容器被攻破 → 只有 appuser 权限 → 无法危害主机
危险等级：✅ 低
```

## ⚙️ 配置文件说明

### Dockerfile 关键改动

```dockerfile
# 1. 定义构建参数
ARG UV_INDEX_URL=https://mirrors.cloud.tencent.com/pypi/simple
ENV UV_INDEX_URL=${UV_INDEX_URL}

# 2. 安装 uv
RUN curl -LsSf https://astral.sh/uv/install.sh | sh && \
    mv /root/.local/bin/uv /usr/local/bin/

# 3. 创建非 root 用户
RUN groupadd -r appuser && useradd -r -g appuser appuser

# 4. 使用 uv 安装依赖
RUN uv pip install --system -r requirements.txt \
    --index-url ${UV_INDEX_URL}

# 5. 设置文件权限
COPY --chown=appuser:appuser . .

# 6. 切换到非 root 用户
USER appuser
```

### docker-compose.yml 关键改动

```yaml
services:
  backend:
    build:
      context: .
      args:
        # 可通过这里配置私服镜像
        - UV_INDEX_URL=https://mirrors.cloud.tencent.com/pypi/simple
```

## ❓ 常见问题

### Q1：构建失败，提示无法连接 uv 官方源？

**A**：修改 `UV_INDEX_URL` 为国内镜像或私服：

```yaml
# docker-compose.yml
- UV_INDEX_URL=http://your-private-server/simple
```

### Q2：容器启动失败，提示权限不足？

**A**：检查主机挂载目录权限：

```bash
# 确保主机目录可写
chmod 755 data logs uploads
```

### Q3：如何回退到旧版本？

**A**：使用 git 回退：

```bash
# 查看历史版本
git log --oneline

# 回退到指定版本
git checkout <commit-hash>
```
